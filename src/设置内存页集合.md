## 设置内存页集合

setup_per_cpu_pageset()
是内核内存管理系统中的一个重要性能优化步骤。它的核心任务是为每个
CPU核芯配置独立的本地缓存页集合（Per-CPU Pageset）

在当前这个阶段，内存管理系统已经初步建立，但为了应对高并发的内存分配需求，内核需要减少
CPU 之间的锁竞争。

在多核（SMP）系统中，如果所有 CPU 都去同一个全局伙伴系统（Buddy
System）里申请内存页，就必须频繁抢占全局自旋锁（Spinlock），这会造成严重的性能瓶颈。如果每个
CPU 维护一个私有的“小仓库”（Pageset），存放少量的物理页，CPU
可以优先从自己的仓库拿货，不需要加全局锁，这样可以减少内存分配竞争。

函数位于文件git/mm/page_alloc.c，其定义为：

```
void __init setup_per_cpu_pageset(void)
{
	struct pglist_data *pgdat;
	struct zone *zone;
	int __maybe_unused cpu;
	for_each_populated_zone(zone)
		setup_zone_pageset(zone);
#ifdef CONFIG_NUMA
	for_each_possible_cpu(cpu) {
		struct per_cpu_pageset *pcp = &per_cpu(boot_pageset, cpu);
		memset(pcp->vm_numa_stat_diff, 0, sizeof(pcp->vm_numa_stat_diff));
	}
#endif
	for_each_online_pgdat(pgdat)
		pgdat->per_cpu_nodestats = alloc_percpu(struct per_cpu_nodestat);
}
```

该函数首先遍历系统中所有包含物理内存的区域（如 ZONE_NORMAL,
ZONE_DMA32），通过函数setup_zone_pageset()在每个区域为每个CPU分配用于存储per_cpu_pageset结构体的内存并初始化该结构体成员，以激活所有区域的页面集。这是最关键的一步，它建立了每个
CPU 申请/释放内存页面的私有快取链表。

如果系统内存结构为NUMA结构（启用CONFIG_NUMA），该函数还要把所有可能 CPU
的启动期页面集（boot_pageset）中的 NUMA
统计清零。在系统启动极早期，内核使用一组临时的启动期页面集。如果不清空这些统计值，它们会干扰后续正式
NUMA 节点的统计，导致内存分配统计（如本地 vs 远程分配）不准确。

最后通过函数alloc_percpu()为每个在线的内存节点（pglist_data）在各自的私有数据区分配用于per_cpu_nodestat
结构体的内存。per_cpu_nodestat 结构体用于记录每个 CPU
在该节点上的内存操作统计信息（例如 vm_stat）。per_cpu_nodestat
结构体驻留在每个 CPU 的私有数据区，更新时无需加锁，性能极高。

这段代码本质上是在“去中心化”。它把原本全局统一的内存统计和分配逻辑，拆分到每个
CPU 自己的私有变量中，从而消除了高并发下的锁争用。

一旦设置完成，内核后续的初始化代码（以及即将启动的用户空间进程）就能通过
get_free_page 以极高的效率获取内存。
