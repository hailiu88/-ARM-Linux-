## 不同类别调度组初始化

传统 CFS 以任务/线程为最小调度单位进行任务调度，以便让每个进程公平地分享
CPU。为了防止某个用户通过创建大量线程来“抢占”其他用户的 CPU
份额，linux运行通过编译开关CONFIG_FAIR_GROUP_SCHED（公平组调度）开启以任务组为单位分配资源的调度策略。这样，系统可以先在不同用户或不同容器控制组（cgroup）之间平分
CPU，然后再在组内部平分。

为了防止实时任务彻底霸占CPU，Linux可以通过编译开关CONFIG_RT_GROUP_SCHED（实时组调度）开启限制实时任务占用的带宽。

在开启了公平组调度的情况下，内核需要为每个
CPU额外维护调度实体和运行队列指针。而在开启了实时组调度后，还需要为每个CPU维护实时调度实体和实时调度队列指针。

位于wait_bit_init()和\#ifdef
CONFIG_CPUMASK_OFFSTACK之间的代码用于在开启了公平组调度和实时组调度后为额外的调度实体和队列分配内存及更新指针值。

内核为每个CPU维护两个cpu掩码（cpumask），用来负载平衡和选择空闲CPU。这些掩码默认会在栈内或结构体内静态分配，但当内核数量较多时（如256），掩码会占用大量的栈空间。掩码占用空间太大会导致栈溢出、cache抖动以及结构体膨胀。为此，Linux设置了编译开关CONFIG_CPUMASK_OFFSTACK。当开启CONFIG_CPUMASK_OFFSTACK后，内核会动态地在堆上为cpu掩码分配内存。位于\#ifdef
CONFIG_CPUMASK_OFFSTACK块之间的代码为每个CPU在其对应的NUMA节点上分配内存。其中kzalloc_node()为在某节点上分配内存并清零，cpu_to_node(i)为获取第i个cpu所在节点编号。

函数init_rt_bandwidth()和init_dl_bandwidth()的作用是初始化全局实时（RT）和
截止日期（DL）调度类的带宽控制结构，防止实时任务耗尽 CPU
导致系统失去响应。其中global_rt_period()指定一个调度周期，global_rt_runtime()指定一个调度周期内实时运行的最长时间。当实时任务超过运行时间，系统就会停止该任务的运行，直到下一个调度周期。

如果要支持对称多核（通过编译开关CONFIG_SMP设置），系统默认会创建一个用于所有
CPU 的全局结构体变量def_root_domain（根调度域）。init_defrootdomain()
的任务就是对这个全局静态变量进行最基本的初始化，包括分配必要的 CPU
掩码（如 span, online, dlo_mask
等），并将该结构体用于引用计数的refcount字段置为1。

def_root_domain是内核管理多核 CPU
负载均衡和实时/限期调度的核心静态基础设施。其默认行为和作用主要体现在以下三个方面：

- 兜底的 CPU 集合管理（Fallback Container）

> 在系统刚启动或用户没有配置 cpuset时，系统默认会将所有的物理
> CPU核芯都划入 def_root_domain 的管辖范围内。

- 全局调度策略的决策中心

> root_domain
> 结构体内置了专门的数据结构，用来加速实时任务的派发。例如cpupri字段（RT
> 优先级管理）记录当前根域内各个 CPU
> 正在运行的实时任务的最高优先级，方便高优先级 RT 任务抢占最闲的
> CPU。cpudl字段（Deadline 截止时间管理）记录各个 CPU
> 上限期任务的最长截止时间，用于实现全局 EDF（Earliest Deadline
> First）调度算法。

- 负载均衡的边界隔离

> Linux 允许通过 cpuset 将 CPU
> 划分成不同的独立集合。此时，内核会动态动态创建（alloc_rootdomain）新的根域。而
> def_root_domain 就作为这套隔离机制的基准模板和初始状态

在Linux任务调度管理过程中，任务是按层级组合的。所有在引导阶段创建的都属于称作root_task_group的根任务组，根任务组是其它任务的“祖先”，所有新创建的进程最初都属于该任务组。

当开启了实时
组调度功能后，函数init_rt_bandwidth()初始化根任务组的实时带宽。它通过
global_rt_period()设置实时调度周期（默认 1s），通过
global_rt_runtime()设置实时运行上限（默认
0.95s），确保根组内的实时进程不会占满用全部的
CPU时间，从而防止系统当机。

Linux允许以组为单位进行任务调度，也允许以进程为单位进行任务调度。编译配置开关CONFIG_CGROUP_SCHED
用以开启控制组调度，作用是允许将进程分组，并以“组”为单位进行 CPU
资源的分配和调度。如果没有开启这个选项，调度器只能以“单个进程”为单位进行调度。

在传统的 CFS
调度中，所有进程都是平等的。这时，如果在一个终端（tty）产生多个进程，比如运行
make -j32 编译内核，系统会产生 32
个进程。假如这时打开浏览器，它只能获得约 \\1/33\\ 的 CPU
时间，导致界面卡顿。为了解决这一现象，Linux采用了autogroup
结构体，通过它，内核可以把同一个 终端的所有进程自动组成一个
任务组。此时，32个编译进程位于一个任务组，而浏览器位于另一个组。两者平等占用
CPU的时间。

autogroup_init位于git/kernel/sched/autogroup.c，作用是初始化默认
autogroup
结构体（名为autogroup_default）。它首先把根任务组赋给autogroup_default的tg字段（task
group），然后把autogroup_default的引用值（reference
count）设为1，接着初始化autogroup_default
结构体中的读写信号量（Read-Write
Semaphore），以便能够使用autogroup_default的读写同步锁，最后通过信号建立指针关联，正式将内核的第一个进程（初始进程init_task）挂载到了已初始化后的默认自动组上。
