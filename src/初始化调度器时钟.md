## 初始化调度器时钟

sched_clock_init() 是 Linux 内核中用于初始化调度器时钟 (Scheduler Clock)
基础设施的关键函数。Linux提供了稳定模式和非稳定模式两版实现，分别用于硬件计数器在所有
CPU核芯之间是绝对同步且频率恒定的系统和x86
等某些存在硬件缺陷的系统。它们的核心区别在于内核是否需要通过软件手段来修复或同步硬件提供的时间戳。当开启CONFIG_HAVE_UNSTABLE_SCHED_CLOCK时，系统采用非稳定版实现，否则，系统采用稳定版实现。

对稳定版实现，函数认为硬件计数器在所有
CPU核芯之间是绝对同步且频率恒定的，sched_clock()
直接读取硬件寄存器并转换，不需要跨核同步，不需要检测时钟漂移。进程从
CPU0 跳到 CPU1，读取的时间戳保证是递增的。

对于非稳定版实现，函数认为不同
CPU核芯的时钟可能不同步（有偏差），或者会随节能状态乱跳。这时，内核维护一个全局的
sched_clock_data，记录每个 CPU 的偏移量。在每次时钟中断（Tick）时，调用
sched_clock_tick() 来校准各核时间，强制让它们对齐。如果 CPU1
读到的硬件时间比 CPU0
之前读到的还要早，内核会通过软件算法强制返回上一次的最大值，防止时间倒流。

对arm版Linux而言，因为它硬件上保证了多核共享同一个 Global Timer
计数器，自然不存在跨核偏差，所以不需要不稳定版中复杂的软件同步逻辑。正因为如此，arm版Linux内核默认状态是采用稳定版实现，即关闭CONFIG_HAVE_UNSTABLE_SCHED_CLOCK。下面我们以稳定版为例介绍函数sched_clock_init()的功能。

函数sched_clock_tick()位于文件git/kernel/sched/clock.c。稳定版的定义为：

```
void __init sched_clock_init(void)
{
	static_branch_inc(&sched_clock_running);
	local_irq_disable();
	generic_sched_clock_init();
	local_irq_enable();
}
```

函数的作用是初始化调度器时间源（sched_clock），并正式启用它。它首先启用静态跳转技术，将NOP指令修改为跳转到sched_clock_running（b
sched_clock_running）指令。这样，sched_clock()函数在运行时会直接跳到sched_clock_running，让代码直接跳到高性能路径（零开销分支），使用高精度的硬件运行，不用再回退到慢速的
jiffies 逻辑。初始化之前，代码沿着NOP之前的路径运行，开销大。

函数local_irq_disable()为禁中断保护。为了确保在设置初始纪元和启动定时器时不会被任何硬件中断打断，从而保证时间轴起始点的原子性。在执行
generic_sched_clock_init
时禁用中断，因为在该函数运行过程中，涉及全局时间基准（epoch）读取，乘 /
移位运算，写入全局变量，而这些运算不是原子的，无法一步完成。如果在初始化还没有完成前发生中断，可能会读到“半初始化”的
sched_clock，导致时间跳变，破坏单调性，所以必须保证初始化过程是原子的。

generic_sched_clock_init()函数主要干三件事：

- 必要时注册 sched_clock（初始化
  mult和shift参数，设置sched_clock读函数）；

- 初始化 epoch_ns和epoch_cyc（时间基准），建立当前 cycle
  对应的纳秒时间；

- 启动一个 hrtimer，周期性防止计数器回绕（wrap）。

它位于文件git/kernel/time/sched_clock.c，其定义为：

```
void __init generic_sched_clock_init(void)
{
	if (cd.actual_read_sched_clock == jiffy_sched_clock_read)
		sched_clock_register(jiffy_sched_clock_read, BITS_PER_LONG, HZ);
	update_sched_clock();
	hrtimer_init(&sched_clock_timer, CLOCK_MONOTONIC, HRTIMER_MODE_REL_HARD);
	sched_clock_timer.function = sched_clock_poll;
	hrtimer_start(&sched_clock_timer, cd.wrap_kt, HRTIMER_MODE_REL_HARD);
}
```

函数首先判断是否需要注册
sched_clock。如果当前还是用jiffy_sched_clock_read（兜底用的 sched_clock
读函数），则利用jiffy_sched_clock_read，通过函数sched_clock_register()更新全局变量cd的核心字段，也就是说从“无时间模型”转向“
mult/shift 时间模型”。

函数update_sched_clock()的作用是更新时间基准。它会更新epoch_cyc和epoch_ns
，其中epoch_cyc通过读函数获取，epoch_ns为基于旧模型算出来当前时间。这样做的目的是让新旧时间无缝衔接，把新时钟“对齐”到旧时间线上。

在更新时间基准后，hrtimer_init()初始化高精度定时器（hrtimer），创建一个基于
CLOCK_MONOTONIC单调时间的高精度定时器。接着设置定时器触发时执行的回调函数sched_clock_poll，最后通过函数hrtimer_start()启动轮询定时器。利用轮询定时器中断，通过周期性调用update_sched_clock()，提前更新epoch_cyc和epoch_ns，让delta
始终在安全范围内，从而保证时间不会倒退。

执行完generic_sched_clock_init()之后，sched_clock_init()
通过关闭中断完成原子初始化，并启用高性能 sched_clock
快路径，为调度器提供可靠的时间基准。
