## 静态关键字的使用方法

要通过静态关键字实现程序跳转的动态切换，首先要通过DEFINE_STATIC_KEY_TRUE(key)或者DEFINE_STATIC_KEY_FALSE(key)定义一个初始值为true或false的静态关键字，在有可能需要切换的位置调用函数static_branch_likely(key)或static_branch_unlikely(key)，如：

```
if (static_branch_likely(&key)) {

    likely code

} else {

    unlikely code

}
```

或

```
if (static_branch_unlikely(&key)) {

    unlikely code

} else {

    likely code

}
```

函数static_branch_enable(&key)把key的enabled域置1，而static_branch_disable(&key)把key的enabled域清零。通过函数static_branch_enable(&key)可以使程序发生跳转，而通过static_branch_disable(&key)函数可以中止程序跳转。如果要改变程序的跳转方向，可以调用static_branch_inc(&key)和static_branch_dec(&key)。函数static_branch_enable()和static_branch_disable()把静态关键字的enabled域设置为1或清零，static_branch_inc()和static_branch_dec()的作用是把enabled域的值加1和减1。当enabled域的减到0时，程序跳转方向就会发生改变。下面为内存管理算法slab用到的为socket分配内存时的一段代码。

```
#ifdef CONFIG_NET
DECLARE_STATIC_KEY_FALSE(memalloc_socks_key);
static inline int sk_memalloc_socks(void)
{
	return static_branch_unlikely(&memalloc_socks_key);
}
#else
static inline int sk_memalloc_socks(void)
{
	return 0;
}
#endif
if (sk_memalloc_socks()) {
	void *obj = cache_alloc_pfmemalloc(cachep, n, flags);
	if (obj)
		return obj;
}
```

当通过编译开关CONFIG_NET选择支持网络功能时，通过DECLARE_STATIC_KEY_FALSE()将memalloc_socks_key的key值定义为伪。因为memalloc_socks_key 默认是 false，sk_memalloc_socks()返回值为false，CPU不进入if内部分支。当系统进入内存回收状态，且需要激活 pfmemalloc 套接字时，内核会调用 static_branch_enable(&memalloc_socks_key)修改memalloc_socks_key的值。此时，sk_memalloc_socks() 运行时返回真，代码进入 if 内部分支，运行cache_alloc_pfmemalloc()分配内存。实际的汇编语言执行流程是：编译出来的汇编在 if (sk_memalloc_socks()) 指令处为一条NOP指令，CPU 直接顺着往下执行，不发生任何条件跳转，完美避免了 CPU 分支预测失败导致的流水线开销。当需要回收内存且激活 pfmemalloc 套接字时，内核利用 text_poke() 等函数，在运行时将原本的 NOP 动态修改为 跳转指令。
