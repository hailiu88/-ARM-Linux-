## 事件跟踪子系统初始化

事件跟踪子系统初始化函数trace_event_init()位于git/kernel/trace/trace_events.c，其定义为：

```
void __init trace_event_init(void)
{
	event_trace_memsetup();
	init_ftrace_syscalls();
	event_trace_enable();
	event_trace_init_fields();
}
```

这段代码是 Linux
内核启动阶段初始化事件跟踪的核心入口，负责搭建起内核观测的基础设施。以下是函数内部关键步骤的详细解析。

event_trace_memsetup()为事件跟踪中用到的两个关键结构体ftrace_event_field
和trace_event_file在内核缓存区分配内存。ftrace_event_field结构体描述了跟踪事件中的各个字段（如字段名、偏移量、大小等）。trace_event_file
核心作用是将事件的静态定义与具体的追踪实例绑定在一起。通过它，内核才能在
/sys/kernel/tracing/events/
目录下为每个事件生成控制文件。它会初始化用于存储 struct
trace_event_call（定义每个事件的数据结构）的列表和哈希表，确保后续事件注册时有地方存放。

init_ftrace_syscalls()是 Linux
内核中函数跟踪器的核心函数，用于初始化系统调用跟踪元数据。它通过for
循环遍历系统调用表中从 0 到 NR_syscalls
的所有系统调用号。该函数首先通过arch_syscall_addr(i) 获取第 i
个系统调用的内核函数入口地址，接着利用该地址获取与地址对应的系统调用元数据地址，从而让
ftrace 能够跟踪特定的系统调用，然后将找到的元数据中的 syscall_nr
字段设置为当前的循环索引 i。如果未启用
CONFIG_HAVE_SPARSE_SYSCALL_NR，或者说系统调用号是连续的，直接将元数据存入数组
syscalls_metadata\[i\] 。如果启用了稀疏编号，则使用 XArray (xa_store)
将元数据存储到名为 syscalls_metadata_sparse 的高效查找树中 。

event_trace_enable()激活事件子系统并将其挂载到
DebugFS/TraceFS。它将所有的 trace_event
注册到全局链表中。执行完这一步后，内核开始在 /sys/kernel/tracing/events/
目录下创建各个子系统（如 sched, irq, kmem）的目录和 enable、format
等控制文件。它还会处理内核启动参数（如 trace_event =
...），实现启动时即开启特定事件的跟踪。下面为该函数的执行过程。

init_ftrace_syscalls 是在准备数据，而这个函数就是在搭建舞台。代码通过
for_each_event 宏，遍历链接程序在编译时存放在 \_\_start_ftrace_events 到
\_\_stop_ftrace_events
段之间的所有事件。它首先通过event_init()对每个事件进行初步初始化，并把初始化成功的事件添加进全局链表
ftrace_events 中。

遍历完所有事件后，利用函数\_\_trace_early_add_events()在文件系统层级结构建立前，先在内存中创建事件的逻辑项。系统需要在
/sys/kernel/debug/tracing
等调试文件系统（debugfs/tracefs）挂载之前，就让顶层跟踪实例（top_trace_array）拥有可用的跟踪点。

拥有可用跟踪点后，函数early_enable_events()根据内核启动参数（如
trace_event=...），在启动早期就开启指定的跟踪事件。

函数最后注册了几个核心的交互组件，分别是：

- trace_printk_start_comm()

> 允许 trace_printk 捕获进程名称，这对于分析跟踪日志非常重要。

- register_event_cmds()

> 注册事件相关的命令。

- register_trigger_cmds()

> 注册事件触发器，允许在事件发生时执行特定动作（如快照、堆栈回溯）。

event_trace_init_fields()初始化事件字段描述符struct
ftrace_event_field的各个字段。每个跟踪事件都有自己的字段（例如进程
ID、指令指针、自定义数据）。此函数遍历所有已注册的事件，解析并初始化它们的字段格式。这决定了当你执行
cat /sys/kernel/tracing/events/xxx/format 时看到的输出格式和定义。
