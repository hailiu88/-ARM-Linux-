## 建立文件系统缓冲区管理机制

在 start_kernel() 的启动链条中，buffer_init()
紧随uts命名空间初始化函数之后。它的核心作用是初始化文件系统的缓冲区管理机制，或者是在给系统准备“文件读写的集散地”。函数位于git/fs/buffer.c，定义为：

```
void __init buffer_init(void)
{
	unsigned long nrpages;
	int ret;
	bh_cachep = kmem_cache_create("buffer_head",
		sizeof(struct buffer_head), 0, (SLAB_RECLAIM_ACCOUNT|SLAB_PANIC|
			SLAB_MEM_SPREAD), NULL);
	nrpages = (nr_free_buffer_pages() * 10) / 100;
	max_buffer_heads = nrpages * (PAGE_SIZE / sizeof(struct buffer_head));
	ret = cpuhp_setup_state_nocalls(CPUHP_FS_BUFF_DEAD, "fs/buffer:dead",
				NULL, buffer_exit_cpu_dead);
	WARN_ON(ret < 0);
}
```

函数的主要任务是调用 kmem_cache_create 创建一个专门存储缓冲区头（ struct
buffer_head
）、名为buffer_head的、内存紧张时可以回收的缓存池。它会根据系统的物理内存大小，计算出允许用于缓冲区的最大内存上限（最多只能占用10%的可用内存，通常由
max_buffer_heads 变量控制），防止文件缓存吃掉所有系统内存。

缓冲区头（Buffer Head）是 Linux 处理磁盘 I/O
的“老员工”。磁盘是以块（Block）为单位存储的，而内存是以页为单位管理的，而buffer_head
就是这两者之间的纽带。它记录了一个磁盘块在内存中的具体位置、状态（是否脏了、是否正在读写）以及所属的设备。

函数的最后会通过函数cpuhp_setup_state_nocalls()注册 CPU
热插拔下线时的回调函数buffer_exit_cpu_dead。当 CPU
被下线时会调用该函数执行清理逻辑。
