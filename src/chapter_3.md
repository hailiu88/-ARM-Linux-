# 第三章	设备树

设备树是一种通过文本语言、以树形结构形式详细描述静态（无法动态发现）硬件系统结构及属性的数据结构。设备树最早由Open Firmware设计，用来作为Open Firmware与客户程序（操作系统）传递数据的通信方法，Linux从版本 3.5.7开始支持设备树。在采用设备树之前，与设备有关的代码以c语言形式固定地写在Linux内核当中，与其它c代码构成一个镜像文件，由U-BOOT在启动系统时导入内存。由于Linux支持大量不同类型的设备，而同一类型的设备又有大量不同厂家，这样，有大量与设备相关的代码要维护，哪怕是一行代码的修改都要重新编译内核程序。为了支持不同的设备，U-BOOT需要利用bootargs给Linux传递大量的参数。
设备树的出现使这一状况发生了巨变。使用设备树后，硬件描述与Linux内核分为两个不同的文件。设备树存在以.dtb结尾的所谓设备blob文件，而内核程序存储在uImage/zImage或FIT文件中。由于硬件信息都包含在设备树文件当中，U-BOOT不再需要向Linux传递大量的硬件信息，而只需要把设备树文件(blob文件)在内存的地址传递给Linux即可。U-BOOT利用r3寄存器(powerPC)或r2寄存器（其它处理器）把blob地址传递给Linux。分为两个文件的另一个好处是当需要支持新的设备或要修改设备参数时，用户只需要编写相应的设备驱动程序，修改设备树文件并编译之，而不再需要再重新编译Linux内核代码。
Linux利用设备树进行三个方面的工作：
-	识别硬件平台
-	动态配置硬件
-	加载设备
