## 上下文跟踪初始化

上下文跟踪系统初始化函数由CONFIG_CONTEXT_TRACKING_FORCE开启。当不启用上下文跟踪功能时，函数context_tracking_init()为空函数。当启用上下文跟踪功能后，context_tracking_init()
初始化内核的上下文跟踪机制，以便区分 CPU 当前是在 内核态 / 用户态 / idle
态，为 RCU、NOHZ、tickless
等机制提供基础支持。它在系统启动的早期阶段被调用，以确保在任何用户态进程运行之前，上下文跟踪机制已经准备就绪。

context_tracking_init()定义在git/ kernel/context_tracking.c，其定义为：

 ```
void __init context_tracking_init(void)
{
	int cpu;
	for_each_possible_cpu(cpu)
		context_tracking_cpu_set(cpu);
}
```
真正开启上下文跟踪的是函数context_tracking_cpu_set()，其定义为：

```
void __init context_tracking_cpu_set(int cpu)
{
	static __initdata bool initialized = false;
	if (!per_cpu(context_tracking.active, cpu)) {
		per_cpu(context_tracking.active, cpu) = true;
		static_branch_inc(&context_tracking_key);
	}
	if (initialized)
		return;
#ifdef CONFIG_HAVE_TIF_NOHZ
	set_tsk_thread_flag(&init_task, TIF_NOHZ);
#endif
	WARN_ON_ONCE(!tasklist_empty());
	initialized = true;
}
```

这段代码展示了 Linux 内核如何开启上下文跟踪。它首先通过宏per_cpu()
访问特定 CPU 的独立变量context_tracking.active，把该 CPU
标记为“上下文追踪处于活跃”状态。然后通过标号跳转技术（static_branch_inc）直接修改内存中的机器码，把
NOP 替换成一条
跳转指令，开启上下文跟踪功能。这是一个非常关键的性能优化手段，它利用了
Static Keys 机制。只要有一个 CPU 开启了追踪，这个全局开关就会加
1。只有当计数 \> 0
时，内核才会在底层汇编入口处（如系统调用入口）通过运行时修改代码（Patching）来激活真正的追踪逻辑。如果计数为
0，则完全跳过，几乎没有性能损耗。由于此时系统还在启动早期，只有 init
任务（PID 0）。通过set_tsk_thread_flag(&init_task,
TIF_NOHZ)给init_task设置 TIF_NOHZ 标志，后续所有 fork
出来的进程都会继承这个标志。这意味着所有进程进入/退出内核时，都会被检查是否需要触发上下文切换通知。
