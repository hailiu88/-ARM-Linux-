## 查找与搜索

find------查找文件命令，格式为：

```
find [查找路径] [匹配条件/选项] [处理动作]
```

常用的选项或匹配条件为：

按名字查找：

-name 按文件名查找（区分大小写）

-iname 不区分大小写

-path 按路径匹配

-regex 正则表达式匹配

示例：

```
$ find /home -name "*.txt"，在/home目录里查找文件名字包含.txt的文件
```

按类型查找：

-type f 普通文件

-type d 目录

-type l 符号链接

-type b 块设备

-type c 字符设备

示例：

    $ find /etc -type d，查找/etc里的目录

按照大小查找：

-size 10k 等于10KB

-size +10k 大于10KB

-size -10k 小于10KB

单位：

c → 字节

k → KB

M → MB

G → GB

示例：

    $ find . -size +100M，查找大小超过100MB的文件

按时间查找：

-mtime 修改时间（天）

-atime 访问时间

-ctime 状态改变时间

写法：

    -mtime 3 \# 3天前

    -mtime +3 \# 超过3天

    -mtime -3 \# 3天以内

示例：

    $ find /var/log -mtime +7，在/var/log查找修改时间超过7天的文件

按权限查找：

    find . -perm 644

按用户或组查找：

    find . -user david

    find . -group users

逻辑组合：

-a 与（默认）

-o 或

! 非

示例：

    $ find . -type f -a -size +1M

    $ find . -name \"\*.c\" -o -name \"\*.h\"

常用动作：

-print（默认） 在屏幕打印结果

-exec 执行命令

```
 find . -name "*.log" -exec rm {} \;，在当前目录查找名字包含.log的文件并执行rm命令删除找到的文件。其中{}表示当前找到的文件，\; 表示 结束命令
```

-delete 删除文件

```
 find . -name "*.tmp" -delete，查找文件名包含.tmp的文件并删除
```

-ok 交互式执行

```
 find . -name "*.log" -ok rm {} \;
```

其它选项还包括：

-maxdepth n 最大查找层级

-mindepth n 最小查找层级

find . -maxdepth 1 -type f

find常用示例：

    $ find /var/log -type f -mtime +7 -delete,，删除7天前日志

    $ find / -type f -size +1G,，查找大文件

    $ find . -type f -name \"\*.sh\" -exec chmod +x {} \\;，修改权限

    $ find . -type f \| wc -l，统计文件数量

**grep**------在文件中查找关键字命令，格式为：

    grep [选项] "关键字" 文件

或与管道结合：

    命令 | grep `选项] \关键字"

匹配控制：

-i 忽略大小写

-v 反向匹配（显示不包含模式的行）

-w 匹配完整单词

-x 匹配整行

-e 指定多个模式

-f 从文件读取匹配模式

示例：

    $ grep -i "error" log.txt

    $ grep -v "debug" log.txt

    $ grep -w "main" test.c

显示控制：

-n 显示行号

-c 统计匹配行数

-l 只显示文件名

-L 显示不匹配的文件

-o 只显示匹配部分

\--color=auto 高亮匹配结果

示例：

    $ grep -n "error" log.txt

    $ grep -c "error" log.txt

递归查找：

-r 递归查找（子目录）

-R 递归查找（包含符号链接）

示例：

    $ grep -r "main" .

上下文显示：

-A n 显示匹配行后 n 行

-B n 显示匹配行前 n 行

-C n 前后各 n 行

示例：

    $ grep -C 3 "error" log.txt

正则表达式模式（默认支持基本正则表达式（BRE））：

-E 使用扩展正则（等同于 egrep）

-F 固定字符串匹配（不使用正则）

-P Perl 正则（部分系统支持）

示例：

    $ grep -E "error\|fail" log.txt

    $ grep -F "a+b" file.txt

常见用法示例：

    $ grep -i "error" /var/log/syslog，查找错误日志

    $ grep -rn "main".，在源码中查找函数

    $ grep -c "error" log.txt，统计匹配次数

    $ ps aux | grep nginx，过滤 ps 输出

    $ grep -o "[0-9]\+" file.txt，只显示匹配部分

正则表达式基础：

表达式 含义

. 任意字符

\* 前一个字符重复

\^ 行首

\$ 行尾

\[abc\] 匹配 a/b/c

\[\^abc\] 不匹配 a/b/c

示例：

    $ grep "^root" /etc/passwd

    $ grep "[0-9]\{3\}" file.txt

使用正则表达式的查找命令：

grep 基本正则

egrep 扩展正则（= grep -E）

fgrep 固定字符串（= grep -F）

现在更倾向于使用：

    $ grep -E

    $ grep -F
