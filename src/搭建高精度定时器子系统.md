## 搭建高精度定时器子系统

高精度定时器（High-Resolution Timers，简称 hrtimer）是 Linux
内核中为了提供纳秒级计时精度而设计的子系统，旨在取代早期仅能提供毫秒级精度（受
HZ 频率限制）的传统时间轮定时器。

传统定时器基于周期性节拍，如果系统 HZ=100，精度只能达到
10ms，如果HZ=1000，精度也只能达到1ms。高精度定时器通过直接编程底层硬件计数器实现，在需要的精确时刻触发中断。它像一块可以随时设定纳秒闹钟的数字表。

与传统定时器的“多级时间轮”不同，高精度定时器在每个 CPU 上维护一个
hrtimer_cpu_base，其中包含多棵按时钟类型分类的红黑树。之所以采用红黑树，主要是为了提高查找效率。内核总是取红黑树最左侧的节点，因为那是最近要到期的任务。

在红黑树上，所有定时器按到期时间排序。
每当新添加一个定时器，如果它的到期时间比当前最左侧的还早，内核会立即重新编程硬件定时器。

系统启动时，高精度定时器并不是立即生效的，它经历两个阶段。在硬件时钟源和时钟事件设备完全准备好之前，高精度定时器运行在模拟模式，精度依然受限于时钟节拍。一旦内核确认硬件支持高精度且稳定，会切换到高精度模式。此时，系统的周期性时钟节拍本身也会由一个高精度定时器来驱动。

在搭建好高精度定时器框架后，系统开发者可以通过hrtimer_init()创建不同的高精度定时器（函数名不包含字母s），其中timer指向保存所定义定时器元数据的hrtimer结构体，which_clock指明该定时器的时钟源，有CLOCK_MONOTONIC
(单调递增时间)、CLOCK_REALTIME系统上与UTC同步的时间、CLOCK_BOOTTIME（包含休眠时间的单调时间）及CLOCK_TAI（国际原子时钟），mode指定该定时器回调函数的运行场景。为了平衡精度和性能，高精度定时器的回调执行分为硬中断上下文和软中断上下文两种场景。对于要求极致精度的任务，直接在时钟中断处理函数中执行（mode值设为HRTIMER_MODE_HARD，默认）。如果mode值设为HRTIMER_MODE_SOFT，而回调函数又可能比较耗时，内核会将其放入
HRTIMER_SOFTIRQ 中处理，以缩短硬中断关闭的时间。

在创建一个高精度定时器后，开发者需要将自己编写的回调函数地址赋予hrtimer结构体的function成员函数，如：

my_timer.function = &my_callback_functio；

函数hrtimers_init()
的功能就是创建能够实现上述高精度定时中断的框架，函数定义在文件git/kernel/hrtimer.c，其定义为：

```
void __init hrtimers_init(void)

{

    hrtimers_prepare_cpu(smp_processor_id());

    open_softirq(HRTIMER_SOFTIRQ, hrtimer_run_softirq);

}
```

该函数是高精度定时器子系统的框架的创立者，为整个内核开启纳秒级计时的能力。它首先利用函数hrtimers_prepare_cpu()初始化当前
CPU（通常是 Boot CPU，即 0
号核）的高精度定时器基础数据结构hrtimer_cpu_base，内核为每个 CPU
分配了一个独立的 hrtimer_cpu_base 结构。各个 CPU
只操作自己的定时器红黑树。

hrtimers_prepare_cpu()通过遍历hrtimer_cpu_base
中的每个clock_base成员，初始化所有时钟基准
的自旋锁，并通过函数timerqueue_init_head()为每个时钟基准设置红黑树根节点（即设置clock_base\[i\]-\>active-\>rb_root），最后会设置
CPU的各种 状态标识。

两个时钟基准对应一种时钟源，如clock_base\[*HRTIMER_BASE_MONOTONIC\]和*clock_base\[*HRTIMER_BASE_MONOTONIC_SOFT\]对应时钟源CLOCK_MONOTONIC，因此每一种时钟源对应两类红黑树，分别用于硬中断和软中断（*hrtimer
结构体包含了红黑树节点timerqueue_node，所以每个定时器本质上就是一个树节点）。所有在该
CPU 上创建的 hrtimer 最终都会根据其类型挂载到对应的 active 树上。

函数open_softirq(HRTIMER_SOFTIRQ,
hrtimer_run_softirq)的功能是正式向内核注册
编号为8的HRTIMER_SOFTIRQ软中断，即把函数指针hrtimer_run_softirq赋予softirq_vec\[8\]的action成员。它告诉内核，“从现在起，如果有人触发了
HRTIMER_SOFTIRQ，请执行 hrtimer_run_softirq 这个函数。”
