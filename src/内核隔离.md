## 内核隔离

内核隔离函数housekeeping_init()位于git/kernel/sched/isolation.c，其定义为：

```
void __init housekeeping_init(void)
{
	if (!housekeeping_flags)
		return;
	static_branch_enable(&housekeeping_overridden);
	if (housekeeping_flags & HK_FLAG_TICK)
		sched_tick_offload_init();
	WARN_ON_ONCE(cpumask_empty(housekeeping_mask));
}
```

这段代码是 Linux 内核 隔离（Isolation）机制
的初始化入口，其核心目的是为“孤岛 CPU”腾挪空间。如果没有在启动参数（如
nohz_full 或 isolcpus）中指定任何隔离需求，函数直接退出。这意味着所有
CPU 都是“管家”，平等分摊内核杂务。

static_branch_enable()作用是性能优化。内核使用 Static Keys
机制优化性能。只有在启用该机制时，内核才会在执行关键路径（如调度或时钟节拍）时切换到带有检查隔离掩码的代码逻辑，避免了在普通模式下不必要的条件判断开销。

如果开启了 nohz_full，系统会尝试关闭非管家 CPU
上的周期性时钟时钟节拍。sched_tick_offload_init()
会初始化专门的内核线程，将这些时钟节拍负载转移到剩下的管家 CPU 上。

WARN_ON_ONCE()作为最后一道防线。系统必须至少保留一个 CPU
来处理诸如内存回收、基数树节点清理等全局事务，否则系统会锁死。
