## 初始化随机数生成器的源

如果使用一个固定的种子（如
0），随机数算法将从同一个数学起点开始计算。这意味着无论你运行多少次程序，第
1 个、第 2 个……第 N
个生成的“随机”数都将保持不变，也就是说，生成的随机数是可预测的。在 Linux
内核源码中，rand_initialize() 是内核 随机数生成器（RNG）
的早期初始化入口，其核心任务是在系统尚未完全启动、缺乏中断熵源的情况下，尽可能使随机数生成器的种子随机，增加随机数的偶然性。函数rand_initialize()定义在文件git/drivers/char/random.c中，其定义为：

```
int __init rand_initialize(void)
{
	init_std_data(&input_pool);
	crng_initialize_primary(&primary_crng);
	crng_global_init_time = jiffies;
	if (ratelimit_disable) {
		urandom_warning.interval = 0;
		unseeded_warning.interval = 0;
	}
	return 0;
}
```

该函数的主要职责是初始化哈希算法结构、采集硬件熵源（CPU
指令）以及混合初始状态。它首先利用函数init_std_data()向输入熵池（input_pool）注入“标准数据”作为初始种子，为内核信赖的随机池准备计算环境。注入的数据有当前时间、架构特定数据（如
CPU 周期计数器、硬件序列号等）、以及指令计数（如
jiffies）。然后它会利用crng_initialize_primary()初始化基于 Chacha20
算法的全局主随机数发生器（CRNG-Cryptographic Random Number
Generator）。它将16个字节的字符串"expand 32-byte k"放入Chacha20
的状态矩阵前 16 字节（必须是这个固定的字符串），然后从输入熵池中提取 48
字节的数据，填充 Chacha20 状态矩阵的剩余部分。这使得 CRNG
不再是空的，而是包含了从系统环境噪声中收集到的初始随机数据。这些是此时最可靠的随机性来源，它包含了硬件随机数、系统高精度计时器（如
TSC 寄存器）以及启动参数。
