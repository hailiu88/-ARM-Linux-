## 建立匿名内存管理机制

在前面我们已经知道，匿名内存是 Linux
内存管理中的核心概念。简单来说，它指没有文件背景的内存映射
。与文件映射相对，匿名内存的内容不是来源于磁盘上的某个具体文件（如代码段、动态库），因此在系统关机或进程退出时，其内容会彻底消失
。

在进程的虚拟地址空间中，以下部分属于匿名内存：

- 通过 malloc() 或 brk() 动态分配的内存堆（heap）。

- 用于存储局部变量和函数调用上下文栈（stack）。

- 存放私有数据，通过 mmap() 带有 MAP_ANONYMOUS 标志分配的内存块。

匿名内存不对应磁盘上的任何文件。当内核需要回收物理内存时，匿名内存不能直接丢弃，必须将其内容写入交换分区（Swap）
才能释放物理页。在 fork()
子进程时，父子进程最初共享相同的匿名物理页。只有当其中一方尝试写入时，内核才会复制出一份新的物理内存页。为了安全，内核在分配匿名页时会将其内容清零，以免读取到前一个进程残留的敏感数据。

由于匿名内存没有文件对应，内核需要一套特殊的机制来追踪“哪个物理页对应哪个进程的哪个虚拟地址”，这就用到了之前介绍的
anon_vma 结构 。当系统需要将某块匿名内存交换到 交换分区时，内核通过
anon_vma
找到所有映射该页的进程，修改它们的页表项（PTE），然后才能安全移出内存。

在 Linux 内核启动过程中，anon_vma_init()
是负责初始化匿名虚拟内存区域管理机制的函数。
其主要任务是为内核的反向映射（Reverse Mapping,
rmap）机制准备必要的内存池和基础结构。函数位于git/mm/rmap.c，其定义为：

```
void __init anon_vma_init(void)
{
	anon_vma_cachep = kmem_cache_create("anon_vma", sizeof(struct anon_vma),
		0, SLAB_TYPESAFE_BY_RCU|SLAB_PANIC|SLAB_ACCOUNT, anon_vma_ctor);
	anon_vma_chain_cachep = KMEM_CACHE(anon_vma_chain,
		SLAB_PANIC|SLAB_ACCOUNT);
}
```

从函数定义可以看出，其主要工作是调用 kmem_cache_create
来初始化两个关键的内存对象池。anon_vma用于存储 struct anon_vma
对象，anon_vma_chain用于存储 struct anon_vma_chain
对象。在前面章节我们已经详细介绍了结构体anon_vma和anon_vma_chain的定义和用处。

anon_vma_init()是内存分配的基础。在该函数执行之前，内核无法高效地管理匿名内存的反向映射。通过运行anon_vma_init()，确保在第一个用户进程创建并请求动态内存分配之前，内存管理子系统的基础设施已经就绪。同时，通过预先建立专用的
Slab 缓存，内核可以在频繁的 fork()
或内存映射操作中快速分配和释放这些管理结构，而不会造成系统范围的内存碎片。
