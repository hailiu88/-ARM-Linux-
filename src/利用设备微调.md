## 利用设备微调

每个设备都有自己的特有信息，如每个设备的Mac地址，设备ID等会不同。利用这些信息，可以增加系统随机数的不确定性，为系统安全多一份保证。

add_device_randomness()利用设备特有的信息（如 MAC
地址、引导参数、UUID）进行最后的微调。这个函数定义在rivers/char/random.c，其定义为：

```
void add_device_randomness(const void *buf, unsigned int size)
{
	unsigned long time = random_get_entropy() ^ jiffies;
	unsigned long flags;
	if (!crng_ready() && size)
		crng_slow_load(buf, size);
	trace_add_device_randomness(size, _RET_IP_);
	spin_lock_irqsave(&input_pool.lock, flags);
	_mix_pool_bytes(&input_pool, buf, size);
	_mix_pool_bytes(&input_pool, &time, sizeof(time));
	spin_unlock_irqrestore(&input_pool.lock, flags);
}
```

它是内核随机数子系统中最基础、最通用的“熵注入”接口，其核心作用是收集设备特征。这个函数不依赖于随机事件的发生时间，而是通过收集硬件设备的唯一标识符或配置信息来增加熵池的不可预测性。数据来源通常是设备的
MAC 地址、序列号、总线
ID、硬件特征位等。确保即使两台完全相同的电脑在同一秒启动，由于网卡 MAC
地址不同，它们的随机数池起始状态也会产生差异。

它组合了高精度的硬件循环计数器（random_get_entropy）和低精度的系统节拍数（jiffies）。即使传入的
buf（如序列号）是固定的，由于执行该函数的时间点会有微秒级的差异，混合后的结果也是唯一的。

函数首先检查主随机数发生器（Chacha20）是否已经完全初始化。在系统启动早期，如果
CRNG 还没准备好，为了让 get_random_bytes
等函数能尽早输出具有一定随机性的数据，哪怕此时还没达到加密安全级别，内核会直接把这些设备数据赋给
CRNG 的状态矩阵，而不只是丢进熵池。

为了调试目的，该函数通过trace_add_device_randomness()添加一个跟踪点，方便开发者使用
ftrace 或 perf 监控到底有哪些设备在贡献随机性。

函数在最后通过_mix_pool_bytes()把启动参数和组合了高精度的硬件循环计数器和低精度的系统节拍数的时钟值混合到输入熵池。
