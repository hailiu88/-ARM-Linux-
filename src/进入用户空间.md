## 进入用户空间

在此之前的初始化工作主要集中在汇编阶段和 C 语言入口 start_kernel()
两个阶段，目标是为 C
运行环境、内存管理和多任务调度打下基础。我们可以把到目前为止所进行的工作大致划分为汇编语言、C语言和过渡三个阶段。

在汇编语言阶段，内核首先执行一段平台相关的汇编代码（如 head.S），进行
硬件基本检查、创建临时页表、设置堆栈，以及清除 BSS 段。

C
语言阶段是内核初始化的核心，它几乎初始化了所有关键的子系统，主要进行了初始化早期控制台、
打印与调试、体系架构初始化、陷阱与中断控制、初始化中断向量表、内存管理初始化、调度器初始化、建立其它核心子系统、VFS
与缓存初始化等。

过渡阶段主要位于rest_init()执行阶段，在这一阶段，创建了 kernel_init
线程和kthreadd线程，启动kernel_init()，主 CPU进入 Idle 状态，变成 0
号进程。

概括来说，在 kernel_init()
之前，内核完成了从裸机环境到受控多任务环境的蜕变，包括内存、调度、中断和基本
I/O 的全面就绪。

此时，系统虽然已经具备了基本的内存和任务管理能力，但由于它还处于内核态，需要通过一系列后续工作过渡到用户态，包括：

- 释放初始化内存

- 激活 SMP 多核处理器

- 设备驱动的全面加载

- 挂载根文件系统

- 开启控制台与打开标准输入输出

- 切换到用户态

kernel_init()函数的主要工作就是完成上述工作。函数位于git/init/main.c，其定义为：

```
static int __ref kernel_init(void *unused)
{
	int ret;
	kernel_init_freeable();
	async_synchronize_full();
	kprobe_free_init_mem();
	ftrace_free_init_mem();
	free_initmem();
	mark_readonly();
	pti_finalize();
	system_state = SYSTEM_RUNNING;
	numa_default_policy();
	rcu_end_inkernel_boot();
	do_sysctl_args();
	if (ramdisk_execute_command) {
		ret = run_init_process(ramdisk_execute_command);
		if (!ret)
			return 0;
		pr_err("Failed to execute %s (error %d)\n", ramdisk_execute_command, ret);
	}
	if (execute_command) {
		ret = run_init_process(execute_command);
		if (!ret)
			return 0;
		panic("Requested init %s failed (error %d).", execute_command, ret);
	}
	if (CONFIG_DEFAULT_INIT[0] != '\0') {
		ret = run_init_process(CONFIG_DEFAULT_INIT);
		if (ret)
			pr_err("Default init %s failed (error %d)\n", CONFIG_DEFAULT_INIT, ret);
		else
			return 0;
	}
	if (!try_to_run_init_process("/sbin/init") ||
	    	!try_to_run_init_process("/etc/init") ||
	    	!try_to_run_init_process("/bin/init") ||
	    	!try_to_run_init_process("/bin/sh"))
		return 0;
	panic("No working init found.  Try passing init= option to kernel. "
	      "See Linux Documentation/admin-guide/init.rst for guidance.");
}
```
