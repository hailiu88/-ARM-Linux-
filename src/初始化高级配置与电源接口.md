## 初始化 高级配置与电源接口

ACPI（Advanced Configuration and Power
Interface，高级配置与电源接口）是一种由英特尔、微软和东芝等公司于 1996
年联合推出的开放式工业标准。
它主要负责操作系统（OS）与系统固件（BIOS/UEFI）以及硬件之间的通信，目的是实现统一的电源管理和硬件配置。
其核心功能包括：

- 电源管理

> 允许操作系统控制硬件的能耗（如进入睡眠或休眠状态）。

- 设备发现与配置

> 操作系统通过 ACPI 表（如
> DSDT、SSDT）识别电脑硬件及其属性，实现即插即用（PnP）。

- 热管理

> 监控系统温度，并根据需要调整风扇转速或降低 CPU 频率以防过热。

ACPI 定义了一系列状态来描述系统或设备的活跃程度。ACPI
标准从全局（Global）和睡眠（Sleep）两个维度来描述电脑的状态。

G 代表 全局状态（Global
State），这是从整机的角度来看的。它定义了用户感知的最大类别。全局状态有：

- G0 (Working)

> 整机正常运行。

- G1 (Sleeping)

> 整机在睡觉，能随时唤醒。

- G2 (Soft Off)

> 软件关机，主板还有微弱供电。

- G3 (Mechanical Off)

> 系统断电。

S 代表 睡眠状态（Sleep State）。因为
G1这个大类别里涵盖了许多情况，所以又细分出了 S
状态来描述主机随眠深度。S状态包括：

- S0

> 对应 G0，机器是醒着的。

- S1/S2

> 浅睡眠。CPU 停止工作，但内存依然满电。

- S3

> 内存通电保留数据，其他全断，唤醒极快，就是我们常说的“睡眠”。

- S4

> 数据存进硬盘，然后彻底断电，唤醒较慢但省，就是“休眠”。

- S5

> 对应 G2，系统关机。

G0 状态下，系统必然处于 S0。G1 状态下，系统可能处于 S1、S2、S3 或 S4
中的一种。G2 状态下，系统处于 S5。

ACPI提供了一个硬件抽象层，使得相同的操作系统可以运行在不同架构的硬件上。由操作系统统一调度电源，比硬件自行管理的
APM 更加精准高效。

ACPI功能通过CONFIG_ACPI启用。如果没有启用，所有与ACPI有关的函数或者为空函数，或者返回值为0或负数。

acpi_early_init()的功能是初始化
ACPI表，以便识别硬件拓扑。如果没有启用ACPI，acpi_early_init()为空函数，定义位于文件git/include/linux/，cpi.h。如果启用了ACPI，函数通过extern在文件git/include/linux/acpi.h中声明。函数定义本身位于文件git/drivers/acpi/bus.c，定义为：

```
void __init acpi_early_init(void)
{
	acpi_status status;
	if (acpi_disabled)
		return;
	printk(KERN_INFO PREFIX "Core revision %08x\n", ACPI_CA_VERSION);
	if (!acpi_strict)
		acpi_gbl_enable_interpreter_slack = TRUE;
	acpi_permanent_mmap = true;
#ifdef CONFIG_X86
	dmi_check_system(dsdt_dmi_table);
#endif
	status = acpi_reallocate_root_table();
	if (ACPI_FAILURE(status)) {
		printk(KERN_ERR PREFIX "Unable to reallocate ACPI tables\n");
		goto error0;
	}
	status = acpi_initialize_subsystem();
	if (ACPI_FAILURE(status)) {
		printk(KERN_ERR PREFIX  "Unable to initialize the ACPI Interpreter\n");
		goto error0;
	}
#ifdef CONFIG_X86
	if (!acpi_ioapic) {
		if (!(acpi_sci_flags & ACPI_MADT_TRIGGER_MASK)) {
			acpi_sci_flags &= ~ACPI_MADT_TRIGGER_MASK;
			acpi_sci_flags |= ACPI_MADT_TRIGGER_LEVEL;
		}
		acpi_pic_sci_set_trigger(acpi_gbl_FADT.sci_interrupt,
				 (acpi_sci_flags & ACPI_MADT_TRIGGER_MASK) >> 2);
	} else {
		acpi_gbl_FADT.sci_interrupt = acpi_sci_override_gsi;
	}
#endif
	return;
 	error0:
	disable_acpi();
}
```

这是 Linux
内核启动过程中的一个关键函数，通常在内核初始化的极早期阶段被调用。它的核心任务是在内存管理系统和设备驱动框架还没完全跑起来之前，先跟主板确认系统配置。

该函数首先通过全局变量acpi_disabled确认内核是否禁用ACPI，如果禁用，直接返回。如果没有禁用，则打印
ACPI
版本，然后通过全局变量acpi_strict确认是否开启宽松模式。如果开启宽松模式，则将变量acpi_gbl_enable_interpreter_slack的值设置为真。变量acpi_permanent_mmap的值用以表示ACPI
表（如 DSDT、FADT）是否永久映射到内核地址空间。

dmi_check_system()是针对X86 特有的补丁，用于修正x86系统的DMI
机器黑名单。它根据 DMI（BIOS 硬件信息）匹配特定机器，对有问题的 BIOS
进行修复。

函数acpi_reallocate_root_table()将 BIOS 中的 ACPI
表复制并重新组织到内核可控内存。该函数通过扫描与校验，再次确认
RSDP（根指针）指向的表是否完整、校验和是否正确。函数通过内核早期的内存分配器，申请一块空间来存放所有
ACPI 表的指针列表，并将所发现的所有表（FADT, MADT, DSDT
等）地址填入这块新申请的空间。这块新申请的空间是新的根索引，存放指向各个
ACPI 子表的物理地址。

函数acpi_initialize_subsystem()初始化 ACPI 全局变量，建立 ACPI
命名空间（ACPI namespace），初始化 AML解释器，以便内核可以执行 ACPI
字节码。AML（ACPI Machine Language） 是 ACPI
规范中定义的一种硬件描述低级伪代码（字节码）。它由符合 ACPI
规范的操作系统所包含的 ACPI 解释器负责解释和执行。

最后一段代码是针对X86系统的特有操作，用于设置SCI （System Control
Interrupt，ACPI专用中断）中断配置。

如果函数在运行过程中出现错误，则调用函数disable_acpi()禁用ACPI功能。

总的来说，函数acpi_early_init()干了三件事，它们是：

- 寻找ACPI Tables表

- 初始化 ACPICA 子系统

- 为后续启动“指路”
