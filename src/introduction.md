
# <center>前言</center>

在计算机系统的世界里，启动（Boot）
是一切运行的起点。从上电的那一刻起，到第一个用户空间进程运行结束，系统完成了一次从"静默硅片"到"可运行操作系统"的完整跃迁。这一过程看似短暂，却凝聚了硬件架构、引导程序、内核初始化、内存管理、调度机制与用户空间体系结构等多个层面的协同工作。然而，对于许多初学者甚至有经验的工程师来说，当电源开启、屏幕闪烁的那一刻，系统内部究竟发生了什么，往往像是一个隐藏在二进制代码下的"黑盒"。

本书的编写初衷，就是为了拆解这个黑盒。我们将带领读者从硬件复位的第一行汇编指令开始，穿过
U-BOOT 的初始化迷宫，目睹 Linux 内核
如何从引导加载程序手中接过指挥棒，直到最终启动第一个进程（Init），拉开用户空间的帷幕。

在嵌入式开发的世界里，ARM 架构与 Linux
操作系统的结合早已成为工业控制、物联网及移动设备的核心支柱。本书将围绕基于ARM的Linux展开，系统梳理从引导加载程序到第一个进程（init）结束的完整技术脉络，

本书的核心内容涵盖了以下四大板块：

第一部分：环境构建与预备（第1-3章）。从 Linux
体系结构入手，详细阐述嵌入式系统的纽带------U-BOOT
引导程序与设备树（Device Tree）机制。这是理解软硬件解耦的关键。

第二部分：内核引导的破晓阶段（第4-14章）。这一阶段被称为内核的"荒芜时期"。我们将深入解析
CPU
模式设置、页表初始化、对称多核（SMP）配置以及命令行参数的逐层解析，见证内核如何在一片空白的硬件上建立起基本的运行秩序。

第三部分：核心子系统的深度构建（第15-25章）。这是本书的硬核灵魂。涵盖了复杂的内存管理（红黑树、内存迁移、规整与合并）、任务调度（CFS
策略、运行队列）、RCU
机制以及中断与时间子系统的建立。此外，还特别探讨了内核安全加固（栈金丝雀、潜伏熵获取）等现代内核安全特性。

第四部分：系统运行与功能闭环（第26-31章）。探讨内核如何迈入多任务状态，建立进程凭证、命名空间与容器能力，并最终启动
init 进程。最后通过操作系统与 Optee 的移植实战，将理论回归于真实平台。

本书的每一章都力求直击本质。通过对跳转标号性能优化、内存加密、死锁检测等高级特性的分析，读者将不仅掌握"如何启动"，更能深刻领悟内核设计的"权衡之道"。

本书重点涵盖：

- 基于 U-BOOT 的启动流程与源码结构

- Linux 内核的引导与初始化阶段

- 各初始化子系统（内存管理、中断、调度、驱动框架等）的调用链分析

- 第一个用户空间进程（init）的创建与执行路径

- 直至 init 生命周期结束的系统行为与机制解析

无论你是希望深入理解系统底层的驱动工程师，还是需要快速搭建环境的应用开发者，本书都旨在为你构建一套完整的知识体系，助你在嵌入式Linux 的丛林中找到清晰的航线。
