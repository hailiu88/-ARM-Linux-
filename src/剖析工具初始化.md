## 剖析工具初始化

3\. profile_init()
初始化老式的内核剖析工具。剖析工具与性能监控的区别在于：这是一个极其轻量级且功能单一的统计工具，主要用于计算内核代码段（.text）的热点。剖析工具通过对指令指针进行抽样来分析内核热点函数。虽然现在多被
perf 取代，但在启动阶段依然保留以支持旧版调试工具。

该函数位于文件git/kernel/profile.c，其定义为：

```
int __ref profile_init(void)
{
	int buffer_bytes;
	if (!prof_on)
		return 0;
	prof_len = (_etext - _stext) >> prof_shift;
	buffer_bytes = prof_len*sizeof(atomic_t);
	if (!alloc_cpumask_var(&prof_cpu_mask, GFP_KERNEL))
		return -ENOMEM;
	cpumask_copy(prof_cpu_mask, cpu_possible_mask);
	prof_buffer = kzalloc(buffer_bytes, GFP_KERNEL|__GFP_NOWARN);
	if (prof_buffer)
		return 0;
	prof_buffer = alloc_pages_exact(buffer_bytes,GFP_KERNEL|__GFP_ZERO|__GFP_NOWARN);
	if (prof_buffer)
		return 0;
	prof_buffer = vzalloc(buffer_bytes);
	if (prof_buffer)
		return 0;
	free_cpumask_var(prof_cpu_mask);
	return -ENOMEM;
}
```

这段源码展示的是 Linux 内核中传统刨析机制（Legacy
Profiling）的内存初始化过程。其逻辑核心为根据内核镜像大小，预留一块内存作为计数器数组。

如果该功能关闭（!prof_on，默认关闭），则直接返回。只有在启动参数中加入
profile=N（N 是缩放因子 prof_shift）时，此函数才会执行实际逻辑。

如果启用刨析机制，则函数首先计算刨析范围（或采样范围）prof_len。\_etext -
\_stext 计算的是内核代码段的总长度。通过右移 prof_shift
位，将内核空间划分为多个区间。每个区间对应一个计数器。

在得出刨析范围后，通过函数alloc_cpumask_var为位图变量prof_cpu_mask分配内存，并用cpu_possible_mask的值初始化该位图。prof_cpu_mask用于内核追踪哪些
CPU核芯参与性能分析，是整个分析机制的“白名单”。

在为prof_cpu_mask成功分配内存后，函数为prof_buffer分配内存。prof_buff的每个单元对应一段代码，单元内存储该段代码执行期间被时钟中断的次数。为了保证在启动阶段能拿到大块连续内存，它首先尝试从
SLAB/SLUB
分配器获取。如果不成功，则尝试分配物理地址连续的内存页。如果还不成功，则利用vzalloc()分配虚拟地址连续但物理地址不一定连续的内存。

如果没有成功获取内存，则释放先前为prof_cpu_mask分配的内存。

profile_init()
初始化的是一个简单的数组。每次时钟中断触发时，中断处理函数会调用
profile_tick()，确定CPU
正在执行的那条指令的内存地址，并判断指针落在哪个区间里，然后在对应区间的计数器加
1。它只能统计内核态，不能统计用户态，只能统计指令执行热点，不能统计缓存缺失或分支预测失败。

在现代 Linux 开发中，perf已经完全取代了这种方式。profile_init
目前更多是出于兼容性保留。
