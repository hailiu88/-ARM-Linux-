## 设备树节点

设备树上的节点可以代表系统上的一个设备，也可以代表系统上的某一子系统或一条总线。描述一个节点的有节点名、节点地址、节点绝对路径和节点标号（label）。

节点名为一个节点的名称，通常以节点所代表的设备名称命名一个节点，其命名方式为name@address，其中name部分为节点名，为1-31个字符的字符串，字符可以从0-9，a-z，A-Z，,
.\_+-中选择。address部分为该节点的地址，地址可以为设备在CPU地址空间的地址，也可以不是CPU地址空间的地址，如何使用由设备驱动程序中设备树处理程序决定。

类似于文件系统的路径，节点的绝对路径是指开始于根节点，以”/”分隔，由节点名(包含地址部分)构成的路径，如在上图中节点cpu0的绝对路径为：

/cpus/cpu0

节点标号是以”:”结尾的字符串，位于节点名的前面，以空白字符分隔。系统可以利用节点标号，也可以利用绝对路径访问节点。

### 根节点

根节点以”/”表示，其绝对路径为”/”，可以拥有#address-cells，#size-cells，model，compatible，和serial-number属性，serial-number为可选项，其余属性为必选项。#address-cells由32位整型数为一个单元构成，可以是一个单元，也可以由多个单元组成。#address-cells指定构成其子节点地址所用的单元个数。#size-cells占用32位，指定表示子节点所占空间所需的单元个数。compatible属性值为字符串，表示与该设备树兼容的设备，通常由代表制造商和设备型号的字符串表示，制造商与设备型号之间以逗号分隔。

根节点下面可以有alias、memory、reserved-memory、chosen、cpus等这几类节点。

### /alias节点

alias节点可以为节点定义别名，通过所定义的节点别名，同样可以访问节点。别名为由1-31个0-9、a-z、及“-”等字符组成，值为要定义别名的节点的全路径，如：

    / {

        aliases {

            serial0 = "/simple-bus@fe000000/serial@llc500";

        };

    };

其中serial0 = "/simple-bus@fe000000/serial@llc500"

为节点serial@llc500定义了一个serial0的别名，用户程序可以通过serial0访问serial@llc500节点。

### /memory节点

memory
节点定义系统的内存结构，每一颗设备树至少要有一个memory节点。如果系统内存由多个不连续的内存空间组成，设备树可以包含多个memory节点。每个memory节点可包含的属性有device-type、reg、initial-mapped-area和hotpluggable等四个属性，其中device-type和reg属性为必选项，其余为可选项。device-type属性的值为字符串“memory”，reg属性为由一个或多个指定地址及空间大小的\<地址
大小\>对构成，hotpluggable属性不需要赋值，当该属性出现时，表明该内存具有热插拔功能。下面是一个具有两个内存区域的memory节点实例：

    memory@0 {

        device_type = "memory";

        reg = <0x000000000 0x00000000 0x00000000 0x80000000>;

        };

    memory@100000000 {

        device_type = "memory";

        reg = <0x000000001 0x00000000 0x00000001 0x00000000>;

        hotplugable

    };

### /reserved-memory节点

表示操作系统不会使用该属性指定的内存。reserved-memory节点通过子节点指明保留区域。reserved-memory节点必须包含#address-cells、#size-cells和ranges三个属性，其中#address-cells和#size-cells的值为32位无符号数，ranges的值为空。

reserved-memory的每一个子节点表示一片保留内存区域。子节点可以包含reg、size、alignment、alloc-ranges、compatible、no-map、resuable等七个属性，所有属性均为可选项，这里，reg属性的编码方式由其父节点的#address-cells和#size-cells指定，size所需的单元数由其父节点的#size-cells决定。alignment属性指明地址对齐值，编码所用单元数由父节点的#size-cells决定，alloc-ranges属性的编码方式与reg的编码方式相同。no-map属性和reusable属性的值为空，当子节点出现no-map属性时，操作系统不能够为该区域生成虚拟映射空间，而当reusable属性出现在子节点时，表明操作系统可以使用该区域，但当器件拥有该区域时，其拥有者可以要回该区域的使用权。下面给出了具有memory和reserved-memory属性的设备树实例。

    / {

        #address-cells = <1>;

        #size-cells = <1>;

        memory {

            reg = <0x40000000 0x40000000>;

        };

        reserved-memory {

            #address-cells = <1>;

            #size-cells = <1>;

            ranges;

            linux,cma {

                compatible = "shared-dma-pool";

                reusable;

                size = <0x2000000>;

                alignment = <0x1000>;

                linux,cma-default;

            };

            display: framebuffer@88000000 {

                reg = <0x88000000 0x800000>;

            };

            multimedia: multimedia@70000000 {

                compatible = "acme, multimedia-memory";

                reg = <0x70000000 0x4000000>;

            };

        };

        fb0: video@12300000 {

            memory-region = <&display>;

        };

        codec: codec@12600000 {

            memory-region = <&multimedia>;

        };

    };

### /chosen节点

chosen节点不是用来指定设备，而是用来向系统传递参数，这类节点可以包含bootargs、stdin-path、stdout-path三种属性，其中bootargs用以向Linux传递引导参数，stdin-path和stdout-path用以指定标准输入和输出。这三个属性的值均为字符串类型，stdout-path和stdin-path的值为用作标准输入和标准输出设备的全路径或phandle，下面的例子给出了/chosen节点的使用方法。

    /{

    chosen {

        bootargs = "root=/dev/nfs rw nfsroot=192.168.1.1 console=ttyS0,115200";

        stdout-path = &uart3;

        stdin-path = &uart3;

    };

    bus@20000000 {

        uart3: uart3@20000000 {

            …

        }

    }

这里，uart3为设备树上某一节点的标号。

### /cpus节点

cpus节点用于表示系统的cpu，根节点必须包含且只能包含一个/cpus节点。cpus节点可以包含多个cpu子节点，而且必须包含#address-cells和#size-cells属性，用以表明其子节点地址的编码方式。cpus节点可以包含其它标准属性，对cpus节点而言，这些属性为可选属性，通常，这些属性为各个子cpu节点所共有。

### /cpus/cpu\*子节点

cpu\*子节点代表一个相对独立的、能够在不影响其它cpu节点的情况下运行操作系统的硬件模块，\*为代表cpu编号的数字。应该把共用同一个MMU的硬件放在同一个cpu节点内，这样，各个cpu节点就相对独立。ARM系统通常有多个相对独立的CPU，比如，i.MX6D具有两个cortex-A结构的CPU，
i.MX6Q具有4个cortex-A结构的CPU，i.MX8具有4个cortex-A72和2个cortex-A53结构的CPU及2个cortex-M结构的CPU，还包含1个DSP和2个GPU，这些CPU能够独立运行，可以放在不同的cpu节点内。

cpu\*节点必须包含device-type、reg、clock-frequency、timebase-frequency等属性，其它可以包含的属性有status、enable-method、cpu-release-addr、power-isa-version、mmu-type等，也可以包含其它标准属性。device-type值为字符串“cpu”，表示该节点为cpu。reg属性值用以表示节点在cpus节点空间的位置，编码方式由cpus的#address-cells和#size-cells决定。Clock-frequency属性的值为一个32位或64位无符号整数，以Hz为单位，表示cpu的运行频率。timebase-frequency属性同样为一个32位或64位无符号整数，表示时间基准采用的频率。Status属性值为字符串，有“okay”和”disabled”两个选项，“okay”表示该节点运行正常，“disabled”表示该节点处于休眠或复位等非正常运行状态。其它属性使用频率没有前面几个的使用频率高，这里不一一介绍，有兴趣的读者可以参考设备树规范文件。下面给出了一个具有两个CPU、采用三级缓存的系统设备树实例。

    cpus {

        #address-cells = <1>;

        #size-cells = <0>;

        cpu@0 {

            device_type = "cpu";

            reg = <0>;

            cache-unified;

            cache-size = <0x8000>; // L1, 32 KB

            cache-block-size = <32>;

            timebase-frequency = <82500000>; // 82.5 MHz

            next-level-cache = \<&L2_0\>; // phandle to L2

            L2_0: l2-cache {

                compatible = "cache";

                cache-unified;

                cache-size = <0x40000>; // 256 KB

                cache-sets = <1024>;

                cache-block-size = <32>;

                cache-level = <2>;

                next-level-cache = <&L3>; // phandle to L3

                L3: l3-cache {

                    compatible = "cache";

                    cache-unified;

                    cache-size = <0x40000>; // 256 KB

                    cache-sets = <0x400>; // 1024

                    cache-block-size = <32>;

                    cache-level = <3>;

                };

            };

        };

        cpu@1 {

            device_type = "cpu";

            reg = <1>;

            cache-unified;

            cache-block-size = <32>;

            cache-size = <0x8000>; // L1, 32 KB

            timebase-frequency = <82500000>; // 82.5 MHz

            clock-frequency = <825000000>; // 825 MHz

            cache-level = <2>;

            next-level-cache = <&L2_1>; // phandle to L2

            L2_1: l2-cache {

                compatible = "cache";

                cache-unified;

                cache-size = <0x40000>; // 256 KB

                cache-sets = <0x400>; // 1024

                cache-line-size = <32>; // 32 bytes

                next-level-cache = <&L3>; // phandle to L3

            };

        };

    };

与缓存有关的属性包括compatible、cache-unified、cache-size、cache-sets、cache-line-size及next-level-cache等，其中，cache-unified、cache-size、cache-sets、cache-line-size表示cache的结构，对cache比较熟悉的读者应该知道其含义，这里不作介绍。next-level-cache代表其下一级cache，上一级cache通过下一级cache的句柄访对其进行访问。
