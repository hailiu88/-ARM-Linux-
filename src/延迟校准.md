## 延迟校准

由于内核在执行某些底层硬件操作（如 I2C
读写、重置外设）时，需要极短且准确的延时（如“等待 10
微秒”），而此时中断可能未就绪或时钟源解析度不够，甚至对某些简单操作，依赖时钟源中断过于复杂，希望只是通过简单循环就能够实现精确延时。内核通过预先测算
CPU 执行简单循环的速度，可以实现这种“靠空转来计时”的功能。

calibrate_delay() 函数的核心任务是测量当前 CPU
在一个系统节拍周期内能执行多少次“忙等待”循环。这个测量结果被存储在变量
loops_per_jiffy (lpj) 中，它是 Linux 内核中 udelay() 和 ndelay()
等微秒级延时函数的换算比例尺。函数位于git/init/calibrate.c，其定义为：

```
void calibrate_delay(void)
{
	unsigned long lpj;
	static bool printed;
	int this_cpu = smp_processor_id();
	if (per_cpu(cpu_loops_per_jiffy, this_cpu)) {
		lpj = per_cpu(cpu_loops_per_jiffy, this_cpu);
		if (!printed)
			pr_info("Calibrating delay loop (skipped) " "already calibrated this CPU");
	} else if (preset_lpj) {
		lpj = preset_lpj;
		if (!printed)
			pr_info("Calibrating delay loop (skipped) " "preset value.. ");
	} else if ((!printed) && lpj_fine) {
		lpj = lpj_fine;
		pr_info("Calibrating delay loop (skipped), " "value calculated using timer frequency.. ");
	} else if ((lpj = calibrate_delay_is_known())) {
		;
	} else if ((lpj = calibrate_delay_direct()) != 0) {
		if (!printed)
			pr_info("Calibrating delay using timer " "specific routine.. ");
	} else {
		if (!printed)
			pr_info("Calibrating delay loop... ");
		lpj = calibrate_delay_converge();
	}
	per_cpu(cpu_loops_per_jiffy, this_cpu) = lpj;
	if (!printed)
		pr_cont("%lu.%02lu BogoMIPS (lpj=%lu)\n", lpj/(500000/HZ), (lpj/(5000/HZ)) % 100, lpj);
	loops_per_jiffy = lpj;
	printed = true;
	calibration_delay_done();
} 
```

该函数先检查当前 CPU
的cpu_loops_per_jiffy是否有值。如果有值，说明当前CPU已经完成过校准，跳过耗时的校准过程。如果没有，则再检查用户在启动参数里是否传了
lpj=xxx。如果传了，直接用，节省启动时间。如果还没有，则检查硬件定时器的频率是否已知且极其稳定。如果是，则直接通过数学公式算出循环次数。如果不是，则利用硬件计数器测量一小段固定位移，看跑了多少圈。如果实测结果为0，则采用穷举校准
（calibrate_delay_converge），通过不断的倍增和折半查找，实际跑循环直到找到能匹配一个
jiffy 的次数。

最终计算出的值会转换成 BogoMips（Bogus MIPS，伪 MIPS）打印在内核日志中。
