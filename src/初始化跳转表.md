# 初始化跳转表
## 初始化跳转表

从前面几节的介绍中我们会发现，在程序编译过程中，编译程序只初始化静态关键字、生成\_\_jump_table，并把与跳转有关的代码放在相应的位置，并没有依据依据静态关键字的值对需要打补丁的指令进行修改。执行指令打补丁的工作由初始化程序完成，完成该任务的函数为jump_label_init()。在完成内存节点和区域初始化工作后，start_kernel()函数调用jump_label_init()，利用存储于\_\_start\_\_\_jump_table和\_\_stop\_\_\_jump_table之间的各个跳转入口结构体及该结构体指向的静态关键字的值，生成NOP指令或跳转指令。如果生成的指令类型为NOP，则把由各个跳转入口的code域指向的指令修改为NOP指令。函数jump_label_init()定义在文件linux/kernel/jump_label.c中，其定义为：

```
void __init jump_label_init(void)
{
	struct jump_entry *iter_start = __start___jump_table;
	struct jump_entry *iter_stop = __stop___jump_table;
	struct static_key *key = NULL;
	struct jump_entry *iter;
	if (static_key_initialized)
		return;
	cpus_read_lock();
	jump_label_lock();
	jump_label_sort_entries(iter_start, iter_stop);
	for (iter = iter_start; iter < iter_stop; iter++) {
		struct static_key *iterk;
		if (jump_label_type(iter) == JUMP_LABEL_NOP)
			arch_jump_label_transform_static(iter, JUMP_LABEL_NOP);
		if (init_section_contains((void *)jump_entry_code(iter), 1))
			jump_entry_set_init(iter);
		iterk = jump_entry_key(iter);
		if (iterk == key)
			continue;
		key = iterk;
		static_key_set_entries(key, iter);
	}
	static_key_initialized = true;
	jump_label_unlock();
	cpus_read_unlock();
}
```

该函数首先利用函数jump_label_sort_entries()把位于\_\_start\_\_\_jump_table和\_\_stop\_\_\_jump_table之间的跳转入口按照其code字段的值的大小由小到大进行排序，之后遍历所有的跳转入口。函数jump_label_type()的作用是依据表
2给出的真值表，通过异或运算，确定补丁指令的类型。函数的定义为：

```

static enum jump_label_type jump_label_type(struct jump_entry *entry)
{
	struct static_key *key = jump_entry_key(entry);  	
	bool enabled = static_key_enabled(key);
	bool branch = jump_entry_is_branch(entry);	
	return enabled ^ branch;
}
```

若静态关键字key的enabled字段的值大于零，static_key_enabled()返回值为1，否则返回值为0。jump_entry_is_branch(entry)返回jump_entry中域key的最低位的值。

函数arch_jump_label_transform_static()的作用是把entry中code字段指向的指令设置为由type类型指定的类型。如果指定的类型为NOP，该函数把NOP指令写入到code指向的地址，如果指令类型为跳转指令，该函数利用entry的code字段和target字段的值，生成正确的跳转指令，然后把生成的指令写入由code域指向的地址。

代码init_section_contains()的作用是检查跳转入口中code指向的地址是否位于初始化代码段（介于\_\_init_begin,
\_\_init_end之间）。如果地址位于初始化段，则利用函数jump_entry_set_init()将跳转入口中的key域的第二位b1置位，表示该跳转入口初始化已经完成。在完成该跳转入口的初始化之后，利用函数static_key_set_entries()把该入口对应的静态关键字中的entries域置为这一跳转入口的存储地址。

程序在遍历跳转标号表格的所有跳转入口后，把全局变量static_key_initialized置为1，表示跳转表格初始化工作已完成。

如果后续程序需要把NOP指令切换为跳转指令，可以通过调用arch_jump_label_transform_static(iter,
JUMP_LABEL_JMP)进行切换。
