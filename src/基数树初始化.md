## 基数树初始化

基数树初始化函数定义在文件git/lib-radix-tree.c文件中，其定义为：

```
void __init radix_tree_init(void)
{
	int ret;
	BUILD_BUG_ON(RADIX_TREE_MAX_TAGS + __GFP_BITS_SHIFT > 32);
	BUILD_BUG_ON(ROOT_IS_IDR & ~GFP_ZONEMASK);
	BUILD_BUG_ON(XA_CHUNK_SIZE > 255);
	radix_tree_node_cachep = kmem_cache_create("radix_tree_node",
			sizeof(struct radix_tree_node), 0,
			SLAB_PANIC | SLAB_RECLAIM_ACCOUNT,
			radix_tree_node_ctor);
	ret = cpuhp_setup_state_nocalls(CPUHP_RADIX_DEAD, "lib/radix:dead",
					NULL, radix_tree_cpu_dead);
	WARN_ON(ret < 0);
}
```

该函数首先使用三条BUILD_BUG_ON宏定义进行编译阶段静态检查。第一条宏确保基数树支持的最大标记数（通常为 3 ）与内存分配标志（GFP Flags，如GFP_KERNEL）所占用的比特位数不超过32。第二条宏保证当前的基数树没有被 IDR机制使用，ROOT_IS_IDR（&~GFP_ZONEMASK获得相应位）用于标识这棵树是否正在被 IDR机制使用。第三条宏确保基数树每个节点的子节点数量不会超过256。

radix_tree_init()真正完成的第一项任务是通过函数kmem_cache_create()为基数树在内核缓冲区分配一块名为radix_tree_node、大小为结构体radix_tree_node所需字节（用以描述基数树上的节点）的内存，并把内存首地址赋予指针radix_tree_node_cachep。

最后，基数树初始化函数通过调用函数 cpuhp_setup_state_nocalls
为基数树注册一个名为lib/radix:dead、在状态为CPUHP_RADIX_DEAD（基数树已死）的回调函数，用以设置CPU热插拔时基数树的状态。CPU上线时，没有回调函数。CPU下线时，执行回调函数radix_tree_cpu_dead，进行资源释放及任务迁移等工作。这里的_nocalls表示注册时不会立刻触发函数回调。
