## 权限与属性管理

**chmod**------ 修改文件权限命令，格式为：

```
 chmod [选项] 模式 文件...
```

常用选项包括：

-R 递归修改目录及子目录文件权限

-v 显示修改过程

\-\-reference=file 将权限设置为参考文件的权限

\-\-changes只显示修改了权限的文件

\-\-silent / --quiet 不显示信息

文件权限包括拥有者权限、组成员权限和其它用户权限，分别用u，g，o表示。权限分读、写和执行三种权限，分别用r，w和x表示。权限也可以用数字表示，具体形式为：

权限 数值

r 4

w 2

x 1

示例：

```
$ chmod u+x \~/bin/repo，表示将用户目录bin里的文件repo的拥有者的权限改为可执行。

$ chmod go-w \~/bin/repo，移除组里其它成员和一般用户对文件repo的写权限。

$ chmod 751 file，将文件file的拥有者权限设为可读写和执行，组内成员的权限设置为可读和执行，不具有写权限，一般用户只有执行权限。
```

**chown**------修改文件或目录的所有者，格式为：

```
 chown [选项] 用户[:组] 文件或目录
```

常用选项包括：

-R 递归修改（目录及其所有子文件）

-v 显示详细修改过程

-c 只显示发生改变的文件

-f 忽略错误信息

\-\-reference=`文件 参照某个文件的 owner/group

-h 修改符号链接本身（而不是它指向的文件）

示例：

```
$ chown david file.txt， 将 file.txt 的所有者改为 david。

$ chown -R david:users /project   /project， 目录及其所有子文件都改为 david:users。
```

**chgrp**------修改文件或目录所属组的命令，格式为：

```
 chgrp [选项] 组名 文件...
```

或

```
 chgrp [选项] 组名 目录
```

常用选项包括：

-R 递归修改目录及其子文件

-v 显示详细处理过程

-c 只显示发生变化的文件

-f 忽略错误信息

-h 修改符号链接本身（而不是链接目标）

\-\-reference=file 参照指定文件的组

示例：

```
    $ chgrp users file1 file2 file3，同时修改file1，file2，和file3所属组

    $ chgrp -R developers /project，递归修改目录
    
    $ chgrp --reference=template.txt newfile.txt，参考文件template.txt进行修改
```
