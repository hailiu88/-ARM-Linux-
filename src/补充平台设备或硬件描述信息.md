## 补充平台设备或硬件描述信息

SFI （Simple Firmware Interface）是由 Intel
开发的一种轻量级固件标准，旨在为那些不需要完整 ACPI
支持的低功耗设备提供硬件描述信息。
ACPI功能强大但极其复杂，需要复杂的解释器来解析固件代码，而SFI非常简单，只提供静态的
C 语言结构体表格，不需要解析字节码。其核心定位是瘦身版的ACPI 。

为了让智能手机、平板电脑等手持设备启动更快、功耗更低，Intel 移除了复杂的
ACPI，转而使用
SFI。它不包含可执行代码，只提供一组静态内存表格。内核只需直接读取这些 C
语言结构体格式的表格，就能知道系统中有哪些 CPU、定时器、I2C 或 SPI
设备。

所有表格都有统一的头部格式，通过扫描物理内存（通常在 0x000E0000 到
0x000FFFFF 之间）的特定签名来定位系统表。它定义了
CPUS、APIC、DEVS（设备表）等多种表格，用于描述中断路由和外设连接情况。核心支持代码在
Linux 内核中仅约 1000 行，远小于 ACPI 的规模。

SFI主要用于 Intel Moorestown 和 Medfield 平台。随着移动市场格局的变化和
UEFI/ACPI 在移动端性能的优化，SFI 失去了存在意义。Linux 内核在 5.6
版本将其标记为过时，并于 5.12 版本正式删除了 SFI 的全部代码。

SFI 是 x86 试图进军超移动领域时产生的一种过渡性技术。它证明了在 x86
架构上也可以不依赖复杂的 ACPI 运行系统，但最终被更标准化的 UEFI/ACPI
生态彻底取代。

sfi_init_late() 是紧跟在 ACPI
子系统初始化之后的一个重要步骤，其核心作用是初始化SFI。函数位于git/drivers/sfi/sfi_core.c，其定义为：

```
void __init sfi_init_late(void)
{
	int length;
	if (sfi_disabled)
		return;
	length = syst_va->header.len;
	sfi_unmap_memory(syst_va, sizeof(struct sfi_table_simple));
	sfi_use_memremap = 1;
	syst_va = sfi_map_memory(syst_pa, length);
	sfi_acpi_init();
} 
```

arch_post_acpi_subsys_init() 负责在 ACPI
就绪后整理硬件信息，而sfi_init_late() 则负责处理那些通过 SFI 表描述而非
ACPI 表描述的硬件。

在一些非标准的 x86 设备上，I2C、SPI
接口的各种控制器（如触摸屏、传感器、电源管理芯片）并不在 ACPI
树里，而是记录在 SFI 的 DEVS 表中。sfi_init_late()会解析 SFI 提供的
MTMR（定时器表）和 RTC 信息，确保系统时钟和中断路由能够正常工作，并将
SFI 发现的设备挂载到对应的内核总线驱动上。

函数首先检查系统是否禁用SFI，如果禁用则直接退出。如果没有禁用，函数先获取
SFI 系统表（SYST）的总长度，然后调用 sfi_unmap_memory
释放掉之前的早期虚拟地址 syst_va，并设置 sfi_use_memremap =
1。告诉SFI功能模块，内存管理系统已经准备就绪。在内核启动的最早期，页表还没完全建立，SFI
使用的是一种临时的、受限的早期映射。

由于内核的内存管理系统已经就绪，所有操作必须使用内存管理系统的地址映射功能。函数通过sfi_map_memory
重新映射物理地址 syst_pa。这样做是为了让 SFI
的表格在内核运行期间以更稳定、更高效的方式驻留在虚拟内存中。

函数最后调用 sfi_acpi_init()与 ACPI 握手。这是一个“兼容性校验”，用于处理
SFI 和 ACPI 同时存在或需要信息交换的特殊场景。
