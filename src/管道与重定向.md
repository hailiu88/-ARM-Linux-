## 管道与重定向

**\|**------管道命令，格式为：

>command1 | command2

作用是将第一条命令command1的输出作为第二条命令command2的输入。

示例：

    $ ps aux | grep nginx

**\>** 及 **\>\>**------输出重定向命令。

示例：

    ls > file.txt，将ls的输出保存到file.txt文件

    ls >> file.txt，将ls命令的输出附着到file.txt的末尾

    ps -ef | grep root | grep ssh | wc -l，多级管道

在 Linux / Unix Shell 中，每个进程默认有 3 个标准文件描述符：

描述符 编号 含义

stdin 0 标准输入（文件或标准设备）

stdout 1 标准输出（文件或标准设备）

stderr 2 标准错误文件（或错误输出设备）

这3个文件可以用描述符表示，也可以用编号表示。利用重定向命令\>或\>\>，可以把标准设备重新定向到其它设备或文件当中。使用格式为：

    command 2 > error.txt，把command的错误输出定向到文件error.txt

示例：

    $ ls not_exist 2 > err.log，把ls命令的错误输出保存到文件err.log

    $ command > out.txt 2>&1，把命令的输出存入文件out.txt，把命令的错误输出定向到标
准输出设备（一般为屏幕）。

& 是一个文件描述符标识符，用于告知 Shell
后面跟随的是一个文件描述符（File
Descriptor），而不是一个普通文件名。如果没有 &，写成 2\>1，Shell
会认为想把错误信息保存到一个名为 1 的普通文件中。

    $ grep keyword file 2>/dev/null，把grep错误定向到黑洞设备，即忽略错误
