## 建立定时器框架

在卸载RCU回调函数后，start_kernel()的下一个任务是通过函数init_timers()建立软件中断框架。init_timers()
的主要任务是建立时间轮（Timer
Wheel）管理机制，并将定时器的具体功能挂载到软中断框架上。它是 Linux
内核定时器子系统的核心初始化入口，主要功能是搭建处理定时任务的基础架构。函数定义在文件git/kernel/time/timer.c里，定义为：

```
void __init init_timers(void)

{

    init_timer_cpus();

    posix_cputimers_init_work();

    open_softirq(TIMER_SOFTIRQ, run_timer_softirq);

}
```

该函数函数内部包含三个步骤。首先，它利用函数 init_timer_cpus()初始化 CPU
本地数据结构，为系统中的每个 CPU 初始化其私有的定时器向量列表。每个 CPU
都有自己的 base ，每个base都会设置 自己的base-\>lock
并初始化用于存放不同超时定时器的“轮盘”，这样可以减少定时器间的竞争。

之后，该函数会利用函数posix_cputimers_init_work() 初始化用于处理 POSIX
CPU
定时器（基于进程/线程执行时间的定时器）的下半部工作。这些定时器不能直接在硬中断上下文运行，因此需要初始化一个
task_work 或延迟处理机制，以便在进程返回用户态前安全执行。

最后，函数调用 open_softirq() ，在内核的软中断向量表中注册
TIMER_SOFTIRQ，并将其处理函数绑定为
run_timer_softirq。每当硬件时钟中断（Tick）发生时，如果发现有定时器到期，内核会触发
TIMER_SOFTIRQ。随后，调度器会在合适的时机运行 run_timer_softirq
来执行那些已经到期的回调函数。

这个函数是时间驱动机制的起点。它建立了一个从“硬件中断”到“软中断处理”的通路。
