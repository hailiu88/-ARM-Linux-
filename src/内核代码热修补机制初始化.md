## 内核代码热修补机制初始化

在内核启动流程中，内核代码热修补机制（Kernel Code Poking /
Patching），是内核在不停止运行的情况下，动态修改自身可执行指令的安全技术。

为了防止攻击者篡改内核，内核的代码段（.text）在内存页表中被标记为
只读和执行。某些情况下，内核需要在运行时根据需要开启/关闭某些功能。为了不影响运行效率，在开启/关闭某些功能时，通常要动态地把NOP指令修改为跳转指令，或把跳转指令修改为NOP指令（采用与静态跳转同样的方法）。热修补机制就是为了解决如何安全地往只读内存里写指令的问题。

由于直接写入只读代码段会触发页面错误，内核采用了一种“绕道”方案。为此，内核创建一个临时的、可写的内存映射，指向与只读代码段相同的物理地址，通过原子替换，将目标位置的指令替换为新指令。这种技术通常只存在于在x86
架构上，对其它架构而言，poking_init()为空函数，在此不作详细介绍。
