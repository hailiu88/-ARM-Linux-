## 创建控制台

前面我们讲到，控制台是内核打印日志和系统信息的接口，函数console_init()的核心任务是让屏幕或串口能打印出日志，以便后续启动出错时开发者能看到报错信息。由于此时内核的很多高级功能（如中断、调度器、完整的内存管理）可能还没准备好，为了尽早打印信息，该函数只做最简单的硬件初始化（比如往串口寄存器写数据），复杂的配置（如中断驱动、高级缓冲区）留到后面。

它是内核启动过程中控制台初始化
的核心函数。该函数位于文件git/kernel/printk/printk.c中，其定义为：

```
void __init console_init(void)
{
	int ret;
	initcall_t call;
	initcall_entry_t *ce;
	n_tty_init();
	ce = __con_initcall_start;
	trace_initcall_level("console");
	while (ce < __con_initcall_end) {
		call = initcall_from_entry(ce);
		trace_initcall_start(call);
		ret = call();
		trace_initcall_finish(call, ret);
		ce++;
	}
}
```

函数首先调用n_tty_init()设置默认的 NTTY 行规程（line
discipline）。它在系统启动阶段执行，使终端设备能够使用默认的行处理逻辑。这是处理输入输出的“规则集”。比如按下回车键，是把它当成
\n 还是 \r\n，这种字符转换逻辑就在这里初始化。

然后循环执行所有存储在代码段\_\_con_initcall_start和\_\_con_initcall_end之间的控制台初始化函数。在编译时，所有通过
console_initcall() 宏声明的驱动函数（如 VGA
驱动、串口驱动），都会被安排在这个代码段。while
循环按顺序执行每一个被注册的控制台驱动初始化函数。一旦某个驱动（比如
uart_init）执行成功，内核的 printk
就能真正把字符显示在屏幕或发送到串口了。

在执行每个控制台驱动初始化函数之前，先调用函数trace_initcall_start()启动初始化调用跟踪功能，记录控制台初始化调用的开始。执行完控制台初始化函数后，调用函数trace_initcall_finish()，记录该控制台初始化函数调用的结束。如果开启了内核调试功能（如
initcall_debug
参数），调用跟踪功能会在日志里记录每个驱动初始化的耗时。这对于优化启动速度非常有用。

这个函数就是把所有注册过的显示/通信设备挨个点亮，确保内核在接下来的启动过程中“有话能说”。在此之前的
printk 信息其实一直缓存在 log_buf
中，直到这一行执行，缓存的信息才会瞬间喷涌到屏幕上。
