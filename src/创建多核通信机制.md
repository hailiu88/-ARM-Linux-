## 创建多核通信机制

在多核 (SMP) 系统中，一个 CPU核芯经常需要让其他
CPU核芯执行特定函数。call_function_init()
的作用是初始化每个CPU核芯用于存放这些远程调用请求的
call_single_queue队列。

函数位于git/kernel/smp.c中，其定义为：

```
void __init call_function_init(void)
{
	int i;
	for_each_possible_cpu(i)
		init_llist_head(&per_cpu(call_single_queue, i));
	smpcfd_prepare_cpu(smp_processor_id());
}
```

在多核 (SMP) 系统中，CPU x有时需要让 CPU y 执行某个特定函数（例如：修改
CPU 1 的寄存器、刷新 CPU 1 的 TLB 缓存），这种机制被称为核间中断函数调用
（Inter-Processor Interrupt Function
Call）。当一个CPU要求另一个CPU运行一个函数时，请求方首先调用
smp_call_function_single(target_cpu, func, ...)。内核会将 func
包装成一个对象，放入目标 CPU 的 call_single_queue。之后，请求方给目标
CPU 发送一个核间中断 (IPI)，触发目标 CPU
中断。目标CPU进入处理程序，清空自己的 call_single_queue
并依次执行里面的函数。

函数call_function_init()初始化的是 Linux 内核中极其关键的
核间调用响应队列。其核心功能是为“发送函数到另一个 CPU
运行”这一机制准备好数据结构。

函数首先遍历系统中所有可能存在的 CPU
，利用函数init_llist_head()为遍历到的每个CPU的无锁链表
call_single_queue初始化了头部。当其他 CPU 想给这个 CPU
发任务时，会将任务挂载到这个 call_single_queue 队列中。

遍历完系统中所有可能存在的 CPU后，利用函数
smpcfd_prepare_cpu()初始化当前启动 CPU（通常是 Boot CPU）的 CFD (Call
Function Data) 结构，确保它现在就能接收和处理来自其他内核的请求。

call_function_init() 是在建立核的内部快递系统。有了它，各个
CPU核芯之间才能互相请求执行函数。

在 start_kernel()
的这个时间点，中断必须处于关闭状态（因为后续紧接着要进行
local_irq_enable()
之前的最后准备）。如果此时中断意外开启了，说明之前的某个初始化函数错误地触碰了中断寄存器，这会导致严重的系统不稳定性。

紧随函数call_function_init()的一行代码WARN(!irqs_disabled(), ...)
用于中断状态最终检查，以确保所有中断处于关闭状态。
