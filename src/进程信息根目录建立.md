## 进程信息根目录建立

在 Linux 中，进程信息伪文件系统根目录（/proc
）并不是一个真正的磁盘目录，而是一个 虚拟文件系统，通常被称为
进程信息伪文件系统。

它的本质是内核数据的窗口。当用 ls 或 cat
查看其中的文件时，内核会实时生成数据，而不是从硬盘读取。它报告进程的详尽运行状态和资源消耗信息，实时报告
CPU、内存、中断等状态，还可以在不重启系统的情况下改变内核行为。

每个 PID在/proc下面都有一个以其PID 命名的PID 文件夹（如
/proc/1234/），存放着该进程的所有秘密，包括：

- 启动该进程的完整命令行参数

- 指向该进程可执行文件的符号链接

- 该进程打开的所有文件描述符

- 进程的状态（运行、睡眠、内存占用、UID/GID 等）

- 进程的内存映射布局（可以看到加载了哪些动态库）

/proc下还有几个文件夹用来保存系统全局信息，包括：

- /proc/meminfo

> 最详尽的内存使用情况（free 命令的数据来源）。

- /proc/cpuinfo

> CPU 的型号、内核数、频率和特性。

- /proc/loadavg

> 系统的负载情况（top 和 uptime 的数据来源）。

- /proc/partitions

> 系统识别的所有硬盘分区。

- /proc/net/

> 网络协议栈的所有实时统计（路由、套接字、流量）。

/proc/sys/用来配置内核参数，这是唯一可以写入的区域（通常需要 root
权限），其下通常包含如下子目录：

- abi

> 包含与二进制应用程序接口（ABI）相关的设置。

- debug

> 存放用于内核调试的开关。通常只有在内核编译了特定调试选项时才会有内容。

- dev

> 包含特定硬件设备及其驱动程序的参数。

- fs

> 管理文件系统的全局参数，如file-max、inode-max 等。

- user

> 控制用户命名空间的相关限制。用于容器技术，限制单个用户可以创建的命名空间数量。

- vm

> 负责虚拟内存管理，如设置脏页回写百分比（dirty
> ratio）、交换分区倾向性（swappiness）、手动释放缓存（drop
> caches）、内存超卖策略（overcommit memory）等。

- net

> 调整网络栈（如开启 IP 转发 net.ipv4.ip_forward）。

- kernel

> 调整内核通用行为（如主机名、内核消息打印级别）。
>
> 函数proc_root_init()负责搭建
> /proc目录树的骨架。函数位于git/fs/proc/root.c，其定义为：

```
void __init proc_root_init(void)
{
	proc_init_kmemcache();
	set_proc_pid_nlink();
	proc_self_init();
	proc_thread_self_init();
	proc_symlink("mounts", NULL, "self/mounts");
	proc_net_init();
	proc_mkdir("fs", NULL);
	proc_mkdir("driver", NULL);
	proc_create_mount_point("fs/nfsd"); 
#if defined(CONFIG_SUN_OPENPROMFS) || defined(CONFIG_SUN_OPENPROMFS_MODULE)
	proc_create_mount_point("openprom");
#endif
	proc_tty_init();
	proc_mkdir("bus", NULL);
	proc_sys_init();
	register_filesystem(&proc_fs_type);
}
```

按照功能逻辑，该函数可以分为四个部分：

- 基础结构初始化

> proc_init_kmemcache()为 proc_inode（/proc 中的文件节点）创建 Slab
> 缓存池，以便高效分配内存。
>
> set_proc_pid_nlink()计算 /proc 根目录的硬链接数，包括了
> /proc/\[pid\]和
> /proc/\[pid\]/task/\[tid\]目录下默认包含的所有文件和子目录（如
> conf、stat、fd 等）。目录的硬链接数通常等于 2 +
> 该目录下的子目录数量（2 代表 . 和 ..）。

- 进程相关结构初始化

> proc_self_init()创建
> /proc/self。当进程访问它时，内核会自动指向该进程自己的 PID 目录。
>
> proc_thread_self_init()创建 /proc/thread-self，指向当前线程。
>
> proc_symlink("mounts", ...)创建一个符号链接，让 cat /proc/mounts
> 实际上读取的是 /proc/self/mounts。

- 核心子目录骨架

> proc_net_init()创建
> /proc/net，存放网络协议栈状态（路由表、套接字等）。
>
> proc_mkdir("fs", NULL)创建 /proc/fs 目录。
>
> proc_mkdir("driver", NULL)创建 /proc/driver 目录。
>
> proc_sys_init()初始化极其重要的
> /proc/sys，这里是修改内核运行时参数（Sysctl）的入口。
>
> proc_tty_init()创建 /proc/tty 相关终端信息。

- 注册文件系统

> register_filesystem()，注册proc类虚拟文件系统。
