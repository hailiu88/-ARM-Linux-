## 设备树源文件(.dts)格式

设备树源文件以.dts为后缀，文件第一行必须为“/dts-v1/;”，v1表示设备树规范第一版。利用编译指令#include
“file”，设备树源文件可以包含以dtsi为后缀的设备树文件。除去没有“/dts-v1/”及#include指令外，dtsi文件的格式与dts文件的格式相同。下面为imx6dl-sabresd.dts文件中部分内容：

```
/dts-v1/;
#include “imx6dl.dtsi”
#include “imx6qdl.dtsi”
/ {
	Model = “freescale i.mx6 DualLite SABRESD Smart Device Board”;
	Compatible = “fsl, imx6dl-sabresd”, “fsl,imx6dl”;
	&battery {
       		offset-charger = <1485>;
        		offset-discharger = <1464>;
        		offset-usb-charger = <1285>;
	}
······
}
```

这里，offset-charger
，offset-discharger和offset-usb-charger是fsl公司定义的属性，使用语法在相应的binding文件中规定。

### 标号

设备树上的节点或属性可以拥有一个标号。标号位于节点或属性前面，由1到31个字符、数字或下划线构成，0-9不能位于起始位。标号的定义方式为在名称后面紧随“：”，通常通过空格把标号和节点或属性分隔。

当要引用句柄或节点的绝对路径时，可以利用标号代替句柄或节点的绝对路径，引用方式是在标号名前加”&”，如上例中的&battery。在扁平化过程中，标号自动转换为句柄，因此，标号名不会出现在blob文件中。

### 节点和属性的定义格式

节点由标号、节点名、单元地址。子节点和属性构成，其中标号和单元地址为可选项，定义格式为：

```
[label:]    node name[@unit address] {
	[label:]    property1 = value;
	[label:]    property2 = value;
	[label:]    child node name[@unit address] {
            ······
	};
};
```

节点既可以包含属性，也可以包含子节点，如果同时包含有属性和子节点，则属性应当位于子节点前面。

属性的定义有两种方式，如果属性需要赋值，则其定义方式为：

\[label:\] property name = value;

属性值可以为位于“”当中的字符串，也可以是位于\<\>当中的由32为无符号整型单元构成的数组，或者是位于\[\]中间的一系列字节串。如果属性值为64位整型数，则可以利用两个32位的单元表示，如\<0x7FFFF0000
0x00000000\>。

属性值可以利用与c语言类似的算术运算符、逐位逻辑运算符、逻辑运算符、关系运算符表示，这些运算符分别为：

算术运算符：+ - \* / %

逐位逻辑运算符 & \| ^ ~ \>\> \<\<

逻辑运算符 && \|\| ！

关系运算符 \> \< == \>= \<= !=

条件赋值运算符 ?: ，格式为

> 条件 ? 条件为真时的取值 : 条件为伪时的取值

属性值可以有多个以“,”分隔的不同的量，如：

Compatible = “fsl i.mx6qdl”, “fsl i.mx8”

可以利用&引用句柄、标号和路径。当位于\<\>中的单元引用在其它节点定义的句柄、节点标号或节点全路径时，编译程序把标号和全路径自动转换为句柄。当在单元阵列\<\>外通过&引用节点时，标号被转换为全路径。在下面的例子中，节点uart@1通过

    uart1: uart@1 {

        interrupt-parent = <&intc1>;

    }

或者通过

    uart1: uart@1 {

        interrupt-parent = <& {/soc/intc@4000}>;

    }

    intc1: intc@4000 {

        interrupt-controller;

    };

等两种方式引用节点intc@4000。

如果属性不需要参数，则通过如下方式为：

\[label:\] properity name;

表示系统支持该特性所代表的功能或设备。
