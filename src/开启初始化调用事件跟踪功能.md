## 开启初始化调用事件跟踪功能

初始化调用（initcall） 是 Linux
内核中一种极为精妙的自动初始化机制。它允许内核在不显式调用每个驱动程序或子系统初始化函数的情况下，按照预定的优先级顺序自动执行它们。我们将在后续章节详细介绍initcall机制。

函数trace_initcall_start_cb的主要作用是开启对每一个初始化函数耗时的监控。当内核启动命令行（cmdline）中加入
initcall_debug
参数时，如果没有开启调式点（TRACEPOINTS_ENABLED），该函数为空函数，不执行任何操作。

如果开启了调试点，initcall_debug_enable()函数通过register_trace_initcall_start()首先将名为trace_initcall_start_cb的回调函数挂载到名为
initcall_start
的跟踪点（钩子）上，并将变量initcall_calltime的地址传递给回调函数。当内核运行到
do_initcall 时，即某个 initcall 被调用时，它会调用
trace_initcall_start()（与do_trace_initcall_start同名）。如果此时已经挂上了钩子函数，内核就会跳转去执行钩子函数，记录时间或打印日志，然后再回来继续执行原来的初始化任务。initcall_calltime用于记录initcall函数开始时间。

挂接完回调函数trace_initcall_start_cb()后，initcall_debug_enable()通过register_trace_initcall_finish把结束时的回调函数trace_initcall_finish_cb()挂接到跟踪点结束跟踪点
initcall_finish（或钩子）上，用以记录某个initcall的结束时间。当某个initcall结束时，会调用trace_initcall_finish()（与do_trace_initcall_finish同名）。用结束时间减去开始时间，得出该初始化函数消耗了多久。

执行一个initcall的函数为：

```
int __init_or_module do_one_initcall(initcall_t fn)
{
    do_trace_initcall_start(fn);     	 // 钩子点，开始
    ret = fn();                    		// 执行真正的初始化函数（如驱动加载）
    do_trace_initcall_finish(fn, ret); 	// 钩子点，结束
}
```

如果说 register_trace_initcall_start 是按下秒表，那么
register_trace_initcall_finish 就是停止秒表并记录成绩。
