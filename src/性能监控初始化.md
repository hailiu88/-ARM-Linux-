## 性能监控初始化

perf_event_init()初始化
性能监控（Perf）子系统，负责在系统启动早期构建性能事件管理所需的基础设施。。它准备好用于追踪
CPU 硬件事件（如 Cache
Miss）和软件事件的数据结构。由于性能监控需要依赖高精度时间采样，因此必须在
time_init() 之后执行。

该函数位于文件kernel/events/core.c，其定义为：

```
void __init perf_event_init(void)
{
	int ret;
	idr_init(&pmu_idr);
	perf_event_init_all_cpus();
	init_srcu_struct(&pmus_srcu);
	perf_pmu_register(&perf_swevent, "software", PERF_TYPE_SOFTWARE);
	perf_pmu_register(&perf_cpu_clock, NULL, -1);
	perf_pmu_register(&perf_task_clock, NULL, -1);
	perf_tp_register();
	perf_event_init_cpu(smp_processor_id());
	register_reboot_notifier(&perf_reboot_notifier);
	ret = init_hw_breakpoint();
	WARN(ret, "hw_breakpoint initialization failed with: %d", ret);
	BUILD_BUG_ON((offsetof(struct perf_event_mmap_page, data_head)) != 1024);
}
```

perf_event_init()
的主要任务是初始化内核用于管理性能计数器（PMC）和软件事件的全局数据结构。它清晰地展示了性能监控子系统
如何从零开始构建其管理框架。

首先，该函数通过函数idr_init()、perf_event_init_all_cpus()和init_srcu_struct()搭建基础设施搭建
，包括初始化整数 ID
管理器、各CPU核芯专有数据结构和可睡眠的读取-拷贝-更新（SRCU）结构。整数
ID 管理器用于Linux内核给每个注册的 PMU（性能监控单元）分配唯一的
ID，方便用户态通过 ID
找到对应的硬件驱动。CPU核芯专有数据结构用于多核并行采样。可睡眠的读取-拷贝-更新机制确保在频繁读取
PMU 列表时，即便有新的 PMU 驱动在加载/卸载，也不会导致系统崩溃或死锁。

随后该函数通过调用函数perf_pmu_register()注册三个软件性质的探测器，分别为perf_swevent、perf_cpu_clock和perf_task_clock，它们不依赖特殊的
CPU 硬件计数器。perf_swevent
用于处理页缺失（page-faults）、上下文切换等内核软件事件，而perf_cpu_clock
和 perf_task_clock则利用内核高精度定时器来提供基于时间的采样能力。

注册软件探测器之后，该函数利用函数perf_tp_register()将性能监控子系统与内核的跟踪子系统关联，让用户可以通过
perf record -e sched:sched_switch 这种命令来追踪内核行为。

接着初始化函数通过函数init_hw_breakpoint()初始化硬件断点支持。这通常用于调试，监控某个内存地址何时被读写（即
gdb 中的硬件观察点 watchpoint）。

最后通过函数perf_event_init_cpu()激活当前启动 CPU 的性能监控功能。

BUILD_BUG_ON()用于编译时检查。它检查 perf_event_mmap_page 结构体中的
data_head 偏移量是否严格等于 1024 字节。这是为了确保内核与用户态（如
perf 工具）通过 mmap
共享内存通信时，数据结构的布局完全一致，否则会导致解析出的性能数据全是乱码。
