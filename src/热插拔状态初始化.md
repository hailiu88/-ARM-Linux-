## 热插拔状态初始化

为了支持热插拔功能，linux在各个cpu的专用存储区定义了一个_per_cpu变量cpuhp_state，其类型为结构体cpuhp_cpu_state，定义为：

```
struct cpuhp_cpu_state {
	enum cpuhp_state	state;
	enum cpuhp_state	target;
	enum cpuhp_state	fail;
	struct task_struct	*thread;
	bool			should_run;
	bool			rollback;
	bool			single;
	bool			bringup;
	struct hlist_node	*node;
	struct hlist_node	*last;
	enum cpuhp_state	cb_state;
	int			result;
	struct completion	done_up;
	struct completion	done_down;
};
```

由于第一个cpu已经开始引导，需要更新该cou热插拔变量cpuhp_state，将热插拔变量中表示器件状态的state域标识为在线。热插拔状态初始化的函数为：

```
void __init boot_cpu_hotplug_init(void)
{
#ifdef CONFIG_SMP
	cpumask_set_cpu(smp_processor_id(), &cpus_booted_once_mask);
#endif
	this_cpu_write(cpuhp_state.state, CPUHP_ONLINE);
}
```

其中，cpumask_set_cpu()函数设置屏蔽字cpus_booted_once_mask中与第一个cpu对应的位，this_cpu_write()把第一个cpu的per_cpu变量cpuhp_state中的state域标识为CPUHP_ONLINE。通过对arm版的this_cpu_write(pcp,
val)函数的分析可以发现，this_cpu_write(pcp, val)函数等价于：

`*（&pcp + _my_cpu_offset） = val;`

而_my_cpu_offset()为从cp15协处理器的TPIDRPRW寄存器读取的值。从上一节我们知道，TPIDRPRW寄存器保存了第一个cpu专用内存的偏移地址，因此，this_cpu_write()把数值写入了第一个cpu的专用变量。
