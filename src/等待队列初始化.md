## 等待队列初始化

sched_init()第一句BUG_ON(&idle_sched_class + 1 != &fair_sched_class
\|\|……)的作用是确保存储调度类的内存没有被污染。如果是对称多核系统（通过CONFIG_SMP设置），还要检查存储调度类stop_sched_class的区域。

任务调度器初始化函数首先通过函数wait_bit_init()函数，调用init_waitqueue_head()函数，初始化全局等待队列（Wait
Queues）的表头和自旋锁。这些队列用于进程的休眠（Wait）与唤醒（Wake
up）机制。这两个函数的定义为：

```
void __init wait_bit_init(void)
{
	int i;
	for (i = 0; i < WAIT_TABLE_SIZE; i++)
		init_waitqueue_head(bit_wait_table + i);
}

#define init_waitqueue_head(wq_head)						\
	do {									\
		static struct lock_class_key __key;				\
		__init_waitqueue_head((wq_head), #wq_head, &__key);		\
	} while (0)

void __init_waitqueue_head(struct wait_queue_head *wq_head, const char *name, struct lock_class_key *key)
{
	spin_lock_init(&wq_head->lock);
	lockdep_set_class_and_name(&wq_head->lock, key, name);
	INIT_LIST_HEAD(&wq_head->head);
}
```

它们分别位于git/include/linux/wait.h、git/kernel/sched/wait.c。链表头初始化函数INIT_LIST_HEAD()定义在git/include/linux/list.h
