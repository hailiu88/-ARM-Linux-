## 创建凭证管理机制

在 Linux 系统中，凭证（Credentials）
管理是内核安全模型的核心，它决定了进程对资源执行的操作类型。在内核层面，凭证管理主要通过
struct cred 结构体实现，遵循 写时复制和
不可变性的原则。每个进程都持有一个指向包含进程所有权限标识的 struct cred
的指针。

Linux
凭证管理遵循严格的生命周期逻辑，以确保权限不会被非法篡改。一旦凭据集被提交，它就不能再被直接修改。如果需要更改权限，必须先使用
prepare_creds()锁定当前进程凭据并克隆一份副本，然后在副本中调整 UID
或能力集，最后使用
commit_creds()将进程的凭证指针指向新副本。原有的凭证会被引用计数机制自动处理。

子进程默认继承父进程的凭证。在执行 exec
加载新程序时，内核会根据文件权限（如 setuid 位）更新新程序的凭证。

Linux内核凭证机制允许进程在容器内拥有根权限（UID
0），但在宿主机看来只是普通用户，实现了权限的完全隔离。

在用户空间，凭证管理更多的是依靠特定的工具和辅助程序，主要有：

- Git 凭证管理器

> 通过 Git Credential Helper 缓存或存储远程仓库的登录信息（如 SSH
> 密钥、Token），避免频繁输入密码。

- Systemd Credentials

> 现代 Linux 发行版通过 systemd
> 机制向守护进程安全地传递密钥、密码等敏感数据，这些数据通常存储在临时内存文件系统（tmpfs）中。

- PAM (Pluggable Authentication Modules)

> 通过 Linux-PAM
> 框架，系统可以灵活配置各种认证方式（密码、指纹、动态口令）来更新进程凭证。

在 Linux 内核启动过程中，cred_init()
负责初始化内核的任务凭据（Credentials）管理机制，其主要任务是为内核中的权限管理建立内存池。函数定义于git/kernel/cred.c，其定义为：

```
void __init cred_init(void)
{
	cred_jar = kmem_cache_create("cred_jar", sizeof(struct cred), 0,
		SLAB_HWCACHE_ALIGN|SLAB_PANIC|SLAB_ACCOUNT, NULL);
}
```

它唯一的任务就是调用 kmem_cache_create ，初始化一个名为 cred_jar
的专用slab内存池。这个池专门用于存储struct cred 结构体。
