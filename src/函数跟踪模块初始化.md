## 函数跟踪模块初始化

函数跟踪模块初始化函数位于git/kernel/trace/trace.c文件中，其定义为：

```
void __init ftrace_init(void)
{
	extern unsigned long __start_mcount_loc[];
	extern unsigned long __stop_mcount_loc[];
	unsigned long count, flags;
	int ret;
	local_irq_save(flags);
	ret = ftrace_dyn_arch_init();
	local_irq_restore(flags);
	if (ret)
		goto failed;
	count = __stop_mcount_loc - __start_mcount_loc;
	if (!count) {
		pr_info("ftrace: No functions to be traced?\n");
		goto failed;
	}
	pr_info("ftrace: allocating %ld entries in %ld pages\n", count, count / ENTRIES_PER_PAGE + 1);
	last_ftrace_enabled = ftrace_enabled = 1;
	ret = ftrace_process_locs(NULL,  __start_mcount_loc,  __stop_mcount_loc);
	pr_info("ftrace: allocated %ld pages with %ld groups\n",
		ftrace_number_of_pages, ftrace_number_of_groups);
	set_ftrace_early_filters();
	return;
 failed:
	ftrace_disabled = 1;
}
```

函数local_irq_save(flags)用于禁止当前 CPU 中断并保存中断状态到flags。

函数ftrace_dyn_arch_init() 与CPU的架构密切相关，用于执行 Dynamic Ftrace
所需的特定 CPU架构初始化操作，主要目的是为所用架构进行指令动态修改（Runtime
Patching）准备环境。早期的函数跟踪代码包括：

- 初始化一些与架构有关的状态——ftrace_dyn_arch_init()。

- 在mcount()或\_\_fentry()入口点写入空操作NOP——ftrace_make_nop()。

- 设置跳转模板——ftrace_make_call()。

- 设置跳转地址——ftrace_update_ftrace_func()。

由于现代硬件大部分都有成熟的函数跟踪机制，而且新的编译器在编译时，所有可跟踪函数的地址已经生成了一张表（位于\_\_mcount_loc 段），函数跟踪模块只需要读取这张表并批量进行ftrace_make_nop。这种“静态表+动态替换”的模式非常通用，不再需要架构层在dyn_arch_init ()里执行复杂的探测，尤其是arm架构指令具有固定的 32位长度，且跳转指令（如bl）的编码格式非常规，因此x86和arm架构的ftrace_dyn_arch_init()均为空函数。

local_irq_restore(flags)函数利用flags变量回复CPU在执行local_irq_save()前的状态，允许cpu中断。

在计算\_\_mcount_loc模块（位于\_\_start_mcount_loc和\_\_stop_ncount_loc之间）的大小后，ftrace_init()函数调用ftrace_process_locs()函数，利用保存在\_\_mcount_loc的地址，将这些位置的call mcount 或 fentry 指令全部替换为 NOP，其主要工作包括：

- 它找到 \_\_mcount_loc 段的起始和结束地址。这张表由编译时工具（如
  scripts/recordmcount）生成，包含了所有可跟踪函数的入口偏移，\_mcount_loc数据格式为结构体ftrace_page

- 对地址进行排序，以便后续快速检索，并剔除重复或无效的入口

- 遍历表中的每个地址，通过函数ftrace_update_code()调用ftrace_nop_initialize()，ftrace_nop_initialize()再调用与架构相关的ftrace_make_nop()或ftrace_init_nop()（如果定义了这一函数，arm64定义了该函数），将原本跳转到跟踪插件的指令替换为
  NOP，并标记代码修改的起止时间

ftrace_init()函数的最后，通过调用set_ftrace_early_filters()函数设置启用函数跟踪时使用的一些过滤参数。
