## 切换EFI固件到虚拟地址

在 Linux 内核启动流程中，efi_enter_virtual_mode() 是连接 EFI 固件环境与
内核虚拟内存环境 的桥梁。EFI（Extensible Firmware
Interface，可扩展固件接口）是位于操作系统与平台固件之间的一种标准接口规范。
它负责在电脑开机时初始化硬件，并引导操作系统启动。

EFI最早由英特尔开发。当 EFI 发展到 1.1 版本后，英特尔将其交给 UEFI
论坛进行标准化，从此更名为 UEFI（Unified
EFI，统一可扩展固件接口）。相比传统 BIOS，UEFI 有如下优点：

- 简化了硬件自检流程，并支持并行初始化，启动速度更快。

- EFI 配合 GPT 分区表可支持超过 2TB 的大容量硬盘。

- 支持鼠标操作和更美观的图形设置界面。

<!-- -->

- 通过 Secure Boot 机制防止恶意软件在系统启动前加载。

在安装了 UEFI 系统的硬盘上，通常会有一个格式为 FAT32 的隐藏分区，即 EFI
系统分区 (ESP)。它存储了系统的引导加载程序（如 Windows 的 bootmgfw.efi
或 Linux 的 GRUB）、硬件驱动程序和系统实用工具。

函数efi_enter_virtual_mode()只用于x86框架，其主要任务是告诉 EFI
固件把固件的运行地址从物理地址切换到分配的虚拟地址。

在内核启动极早期，EFI
运行在物理地址空间。一旦内核开启了内存管理单元，物理地址就不再直接可用。EFI
提供了一些在系统运行时依然需要的服务（如修改 EFI
变量、重启系统、设置硬件时钟）。为了能够继续使用固件提供的这些服务，该函数会调用
EFI 运行服务中的
SetVirtualAddressMap，将固件的代码和数据映射到内核的虚拟地址空间，确保内核在保护模式/长模式下依然能安全地调用这些固件函数。

该函数会遍历 EFI 内存图，寻找标记为 EFI_RUNTIME_SERVICES_CODE 或 DATA
的区域，并在内核页表中为它们建立映射。如果内核开启了 MMU
但没有执行这个切换，后续任何对 EFI 服务（比如读写 UEFI
启动项）的调用都会因为访问非法的物理地址而导致内核直接崩溃。

efi_enter_virtual_mode()位于git/arch/x86/platform/efi/efi.c。由于该函数只用于x86框架，这里不作更多介绍。
