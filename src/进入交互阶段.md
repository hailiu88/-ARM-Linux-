## 进入交互阶段

在此之前，内核不能够也没必要被打断，内核运行在完全确定性的单线逻辑中。然而，操作系统需要接收操作者的指令及对外部发生的事件进行响应，只有开启了中断响应，计算机才能够响应用户输入。

函数local_irq_enable()第一次全局开启硬件中断，并开启中断追踪。该函数通过宏定义的方式定义，位于文件git/include/linux/irqflags.h，其定义为：

```
#define local_irq_enable()				\
	do {							\
		trace_hardirqs_on();			\
		raw_local_irq_enable();			\
	} while (0)
```

raw_local_irq_enable()的作用是执行底层的硬件指令（如 x86 上的
sti）来真正开启当前 CPU 的硬中断响应。带 raw\_
前缀表示这是一个底层的、不带任何额外调试或追踪开销的原始操作。
raw_local_irq_enable()的真正定义是arch_local_irq_enable()，是通过在同一文件的下列宏定义实现的：

```
#define raw_local_irq_enable() arch_local_irq_enable()
```

arch_local_irq_enable()与硬件架构密切相关，其定义通常为汇编语言，文件位于架构对应的目录下。对32位arm而言，函数定义位于git/arch/arm/include/asm/irqflags.h文件，以GCC内联汇编的方式嵌入了开启中断的汇编语句。

```
static inline void arch_local_irq_enable(void)
{
	asm volatile(
		"	cpsie i			@ arch_local_irq_enable"
		:
		:
		: "memory", "cc");
}
```

volatile告诉编译器不要优化此段代码。该段代码表示执行汇编指令 cpsie
I（开启中断指令），无输出变量，无输入变量，该指令可能会改变内存的值，可能会影响状态寄存器的值。

GCC内联汇编的格式为：

```
 asm(code : output operand list : input operand list : clobber list);
 ```

这是系统启动以来第一次全局开启硬件中断。在此之后，硬件（如时钟、键盘、网卡）可以随时打断内核执行。由于中断开启，之前初始化的
timer_probe 开始产生 时钟节拍，驱动 profile_tick() 等函数运行。
