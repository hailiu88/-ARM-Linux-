基于ARM的Linux操作系统引导详解

标准属性

设备树规范定义了一些标准规范。标准规范定义了大部分设备树都需要的属性，设备树编写者只要按照这些标准属性的使用方法使用即可。除了这些标准属性外，设备驱动程序编写人员还可以定义自己的属性，当然，这些属性只能供自己在编写设备驱动程序时使用，或供其它同类设备驱动程序编写人员使用。

在描述属性时经常要用到一个叫做单元的概念，在设备树中，一个单元是指一个32位的无符号整数。

compatible属性

Compatible的值为字符串列表，通常以制造商、型号对的形式出现，《制造商、型号》对在字符串列表出现的次序为属性越专用，越先出现，越通用，越靠后出现。Linux内核利用该值选择与设备兼容的驱动程序。字符串必须为小写字符、数字0-9和破折号，第一个字符为小写字符，如

compatible = “fsl, mpc8641” , ”ns16550”

model属性

model属性的值为字符串，用以表示器件的型号，通常应采用制造商，型号的形式，如：

model = “fsl, MPC8349EMITX”

phandle属性

Phandle属性的值为32位无符号整数，当一个节点设置了phandle属性，可以通过该phandle的值访问该节点，比如，当采用如下方式定义节点pic的属性：

pic@10000000 {

phandle = <1>;

interrupt-controller;

reg = <0x10000000, 0x100>;

};

时，它定义了一个值等于1的phandle。这时，其它节点可以利用1引用pic节点，如

regulator@1 {

interrupt-parent = <1\1>;

}

表示regulator@1的中断控制器为pic@1000000。

status属性

status属性用来指示器件的运行状态，status的值只能是okay，disabled，reserved，failed，failed-sss当中的一个，其中sss为错误代码。当节点缺省status属性时，其默认值为okay。如果基于已有的设备树编写一个新的系统的设备树，可以将新系统中不支持的器件节点的status属性置为”disabled”，这样，系统在启动时就不会查询该设备。

#address-cells和#size-cells属性

一个节点的#address-cells和#size-cells属性定义访问其子节点的方法，设备树以32位作为一个单元，#address-cells定义子节点reg属性中用多少个32位单元定义子节点地址，#size-cells定义子节点reg属性中用多少个单元定义子节点的空间大小，比如，若节点中#address-cells和size-cells分别为：

address-cells = <2>;

size-cells = <1>;

则其子节点中reg属性应该用如下方式定义：

reg = <0x10000000 0x00000008 0x100>

其中0x1000000000000000表示该子节点地址，要用两个单元表示，该子节点占用的空间大小为0x100，表示该空间大小时需要采用一个单元。

reg属性

reg属性表示节点地址及该节点空间大小，其中用于表示地址的单元数和表示空间的单元数由其父节点的#address-cells属性和#size-cells属性决定。如果子节点占用多个不连续的地址空间，可以用多组地址、大小组合表示，比如，一个节点占用了从0x10000000开始的100个地址和从0x200000000开始的200个地址，又假设该节点的地址和大小均用一个单元表示，则该节点的reg属性可以表示为：

reg = <0x10000000 100 0x20000000 200>;

如果#size-cells = <0>，reg当中表示大小的部分可以省略。

ranges 属性

ranges属性用来定义子节点到父节点之间的地址转换，其格式为：

ranges = <[子节点地址 父节点地址 范围] [子节点地址 父节点地址范围]…>

其中，表示子节点地址所需的单元数由父节点内的#address-cells决定，也就是range属性所在的节点内的#address-cells的值决定，父节点地址所用单元数由父节点上一级的#address-cells的值决定。若range的为空，则子节点和父节点地址空间相同，在下面的例子中，

bus{

    #address-cells = <2>;

    soc {

    compatible = "simple-bus";

    #address-cells = <1>;

    #size-cells = <1>;

    ranges = <0x0 0xe0000000 0x00000000 0x00100000>;

    serial@4600 {

        device_type = "serial";

        compatible = "ns16550";

         reg = \<0x4600 0x100>;

        clock-frequency = <0>;

        interrupts = <0xA 0x8>;

        interrupt-parent = \<&ipic\>;

        };

    };

}；

soc内的#address-cells的值为1，所以在ranges定义中，serial地址占用一个单元，而在bus节点内，#address-cells的值等于2，所以父地址(soc地址)占用两个单元，即，soc地址为0xE000000000000000，这样，serial在soc中的地址为0xE000000000004600。

dma-ranges属性

与ranges属性格式相同，用于把dma空间地址映射到其父空间地址。

interrupts属性

interrupts属性用于能够产生中断的器件，该属性没有标准的定义方式，一般与器件绑定，由设备提供方或驱动程序编写者定义，相应的绑定文件通常可在Documenntation/devicetree/bindings目录里找到。大部分情况下，interrupts属性的值由中断号和中断方式（上升沿/下降沿/电平）组成，如：

interrupts = <0xA 1>;

该值的意义可以解释为中断号10，上升沿中断，当然也可以有其它解释，如何解释由该中断的绑定文件定义。

interrupt-parent属性

interrupt-parent属性用以定义中断的上一级中断控制器。目前，有许多中断控制器结构是分层实现的，interrupt-parent属性可以反映这样的层次结构。如果中断的上一级控制器节点定义了句柄，则interrupt-parent的值是在上一层控制器定义的句柄，如果它的上一级控制器节点定义了标号，则interrupt-parent属性的值可以利用所定义的标号。下面给出了利用句柄和标号作为interrupt-parent属性值的实例。

若在中断的上一级控制器节点有phandle = <1>;

则interrupt-parent属性可定义为：

interrupt-parent = <1>;

如果中断的上一级控制器节点定义了标号intc1:，则interrupt-parent属性可定义为：

interrupt-parent = <&intc1>;

interrupt-cells属性

该属性用于中断控制器，由于指明该控制器控制范围内中断属性，也就是上面介绍的interrupts属性占用的单元数，在3.2.9的例子中，中断属性占用两个单元。

interrupt-controller属性

该属性作用于中断控制器，用于说明属性所在的节点为中断控制器。interrupt-controller属性不需要值，其定义方法为：

intc1: intc@100{

interrupt-controller;

}

关联(Nexus)节点

所谓关联节点是指具有#specifier-cells属性的节点，其中#specifier-cells属性定义specifer占用的单元数。specifier通常用来指示一个模块，如#gpio-cells中的specifier为gpio，对应一个GPIO模块。而#specifier-cells的值表示指定specifier内的一个单位时需要的单元数。如果一个节点有#gpio-cells = <1>；属性定义，则该节点为一个关联节点，指示符占用一个单元。

下面以imx6q gpio1模块为例说明关联节点的使用方法。

在支持imx6q/imx6dl 的Linux设备树文件imx6qdl.dtsi中，gpio1节点的定义为：

gpio1: gpio@209C000 {

compatible = “fsl, imx6q-gpio”, “fsl, imx35-gpio”;

reg = <0x0209C000 0x4000>;

interrupts = <0 66 IRQ_TYPE_LEVEL_HIGH>, <0 67 IRQ_TYPE_LEVEL_HIGH>;

gpio-controller;

gpio-cells = <2>;

interrupt-controller;

#interrupt-cells = <2>;

};

在上面的例子中，interrupts属性说明该节点能够产生中断，interrupts有三个单元，说明其父节点的#interrupt-cells=<3>，即，其父节点要指示一个中断要用3个单元。对imx6q而言，第一个单元用于表示gpio模块编号，第二个单元为中断编号，第三个单元说明中断类型。

gpio-controller属性说明该模块为gpio控制模块，驱动程序可以依据该属性判断该模块具有gpio控制功能。

#gpio-cells =<2>表示要指定该模块的一个单位需要使用两个单元，第一个表示gpio管脚编号，第二个编号表示gpio状态，比如ACTIVE_LOW。

interrupt-controller属性说明该gpio模块还是一个中断控制器，#interrupt-cells=<2>说明要指定该中断控制器所控制的中断需要两个单元，第一个单元指定中断编号，第二个单元指明中断类型。由于已知中断发生在gpio1内，因此不需要象其父节点一样要指定gpio编号。

此外还有<specifier>-map、<specifier>-map-mask、<specifier>-map-pass-thru等属性，使用范围没有#<specifier>-cells广，这里不作介绍，可以参考设备树规范。

上面介绍了少量标准属性，设备树标准只定义了少量属性。由于设备树非常灵活，而自定义属性并不会影响其它设备运行，设备树使用了许多供应商自定义属性。
