基于ARM的Linux操作系统引导详解

针对框架的中断初始化

中断初始化的第二步由函数init_IRQ()实现，是中断子系统初始化的真正执行者。由于 Linux 支持众多处理器架构，init_IRQ() 的具体实现千差万别，因而，该函数通常位于 arch/ 目录下（如 arch/arm/kernel/irq.c）。尽管实现各异，但其核心逻辑通常包含以下三步。

第一步是调用中断控制器驱动，这是最关键的一步。它会调用 irqchip_init()扫描 __irqchip_of_table 段，寻找匹配设备树的中断控制器。这里就是 IRQCHIP_DECLARE 宏注册的初始化函数真正被执行的地方。

第二部设置异常向量表，初始化 CPU 侧处理中断的入口点。例如在 ARM 架构中，会将中断向量表设置到 VBAR 寄存器中，确保当引脚电平变化时，CPU 知道该跳转到哪行汇编代码。

最后一步初始化特定架构的中断栈，为每个 CPU 分配专门的中断栈，防止中断处理过程中的栈溢出影响到当前运行的进程栈。

下面我们以ARM的gic为例介绍中断系统的初始化过程及功能。

init_IRQ()函数定义在git/arch/arm/kernel/irq,c文件里，其定义为：

void __init init_IRQ(void)
{
	int ret;
	if (IS_ENABLED(CONFIG_OF) && !machine_desc->init_irq)
		irqchip_init();
	else
		machine_desc->init_irq();
	if (IS_ENABLED(CONFIG_OF) && IS_ENABLED(CONFIG_CACHE_L2X0) &&
	    (machine_desc->l2c_aux_mask || machine_desc->l2c_aux_val)) {
		if (!outer_cache.write_sec)
			outer_cache.write_sec = machine_desc->l2c_write_sec;
		ret = l2x0_of_init(machine_desc->l2c_aux_val, machine_desc->l2c_aux_mask);
		if (ret && ret != -ENODEV)
			pr_err("L2C: failed to init: %d\n", ret);
	}
	uniphier_cache_init();
}

该函数是 ARM 32位架构 特有的 init_IRQ 实现。它体现了 Linux 内核从老式板级描述符（Machine Desc）向设备树过渡时期的逻辑判断。该函数功能分为初始化中断控制器、初始化 L2 Cache、初始化特定平台缓存等三部分，以完成架构级中断子系统初始化。

函数首先初始化中断控制器（irqchip）。由于部分系统还没有通过设备树配置系统硬件，arm版Linux系统支持通过板级描述符和设备树两种方式描述系统的硬件组成。利用板级描述符和设备树描述的中断控制器的初始化采用不同的代码。

函数的第一部分决定了使用哪种方式来驱动中断控制器。如果开启了设备树（CONFIG_OF）且没有定义特定板级的初始化函数，内核会调用通用的irqchip_init()函数初始化中断控制器。它会扫描所有用 IRQCHIP_DECLARE 声明的驱动，并与设备树节点匹配。设备树 + irqchip 框架是现代 SoC（如 i.MX6, RK3288 等）的标准做法，也是现代 ARM 推荐方式，它完全摆脱了板级硬编码。

如果板级描述符提供了machine_desc->init_irq()，则调用板级中断初始化函数。这是为了兼容老旧设备。在没有设备树的年代，每款开发板都在 arch/arm/mach-xxx 下通过宏 MACHINE_START定义了machine_desc 结构体，并在其中硬编码 init_irq 函数指针。

irqchip_init() 是设备树驱动方式初始化中断控制器的入口函数，主要职责是扫描并初始化系统中所有的中断控制器。它会解析设备树中 interrupt-controller 节点，根据 compatible给出的字符串内容匹配中断控制器驱动，调用对应 irqchip 的初始化函数，建立 irq_domain，设置 irq_desc->handle_irq，最后注册硬件中断处理入口。如果内核配置中未开启 CONFIG_IRQCHIP，则该函数会被定义为一个空的内联函数，以保持代码兼容性。

irqchip_init()位于文件git/drivers/irqchip/irqchip.c里，其定义为：

void __init irqchip_init(void)
{
	of_irq_init(__irqchip_of_table);
	acpi_probe_device_table(irqchip);
}

这段代码展示了现代 Linux 内核（通常是 ARM64 或支持 ACPI 的 ARM32）中 irqchip_init 的标准实现。它的核心任务是通过设备树（DT）或 ACPI 表来发现并初始化硬件中断控制器。

of_irq_init(__irqchip_of_table)是针对 设备树 (Device Tree) 系统的初始化入口。如前所述，__irqchip_of_table存储了所有通过 IRQCHIP_DECLARE 宏声明的驱动程序以及与初始化回调函数关联的compatible 字符串。它的每个条目都包含一个指明兼容中断控制器的字符串和与之对应的初始化回调函数。of_irq_init()会遍历设备树，寻找带有 interrupt-controller 属性的节点（设备树处理函数名称通常以of_开始）。of_irq_init 还会解析设备树中的中断层级。它会先初始化根中断控制器（如 GIC），因为其他设备（如 GPIO 控制器、DMA）的中断线是连接在它之上的。只有父节点初始化成功后，子控制器才会执行。

acpi_probe_device_table(irqchip)是针对 ACPI (Advanced Configuration and Power Interface) 系统的初始化入口，常用于服务器级 ARM 或 x86 架构。它会扫描 ACPI 表（如 MADT - Multiple APIC Description Table），寻找匹配 irqchip 类型的条目。在没有设备树的情况下，它通过固件提供的数据结构来配置中断路由。

较新的内核也可能包含 IRQCHIP_ACPI_DECLARE ，以支持通过 ACPI 表进行硬件探测。 在不支持设备树的系统（如 ARM64 服务器或部分 x86 环境）中，IRQCHIP_ACPI_DECLARE用于将中断控制器的初始化函数与 ACPI 表中的特定类型关联起来。它利用 GCC 属性将初始化条目放入一个名为 __irqchip_acpi_table 的特殊段中。函数acpi_probe_device_table() 会遍历这个段，并与 ACPI 表中的 ID 进行匹配。

函数的第二部分处理 ARM 架构中常见的 L2C-310 (PL310) 或类似的二级缓存控制器，它通过函数l2x0_of_init()初始化PL310。L1 缓存通常在 CPU 内部，而 L2 被视为外部缓存（Outer Cache），需要通过特定的寄存器操作和同步指令来管理。l2x0_of_init()使用板级描述符machine_desc，它 提供了 L2 配置参数。

函数的最后一步工作是初始化特定平台缓存（如 UniPhier）。 uniphier_cache_init()是针对 索喜（Socionext）UniPhier 平台 的特殊补丁。这属于厂商特定的钩子函数，用于处理该平台特有的缓存控制器。
