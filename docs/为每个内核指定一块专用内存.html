基于ARM的Linux操作系统引导详解

为每个内核指定一块专用内存

专用内存分配策略

由于Linux操作系统采用地址映射表进行虚拟地址与物理地址的转换，期间要不断地访问页表。为了提高内存访问效率，Linux在分配内存时通常会设置一个基本（最小）单位，而不是以字节为单位进行内存分配。采用的最小单位不同，内存的使用效率就不同。最小单位一般为页。

各个cpu专用内存主要包括介于_per_cpu_start与_per_cpu_end间的、由编译与链接程序生成的静态内存。各个cpu专用内存，用于可导入模块的保留内存，以及一部分用于动态使用的内存，这三部分合起来为统称为最小分配内存。由于在进行内存分配时有最小分配单位（atom_size），实际分配的内存必须为最小分配单位的整数倍，因此实际分配的内存数量（alloc_size）一般要大于最小分配内存。

对于单核系统，利用最小分配单位进行内存分配时内存的利用率可用达到最高。对于具有NUMA内存结构的系统，在一个NUMA节点的内存区间内为访问该节点的所有CPU分配专用内存。这时如果使用最小分配单位为其中的每个cpu分配内存，可能会浪费比较多的内存。比如有7个cpu访问某个节点，每个cpu需要10k内存，这时如果最小分配单位为4k，每个cpu需要3个分配单位的内存，7个cpu共需要21个分配单位的内存，即84k内存，而7个cpu实际只需要70k内存，有14k内存的浪费。事实上，只需要为7个cpu分配72k内存即可满足分配数量为最小分配单位整数倍的要求。在为NUMA结点所对应的每个cpu分配内存时，通过把分配单位降到2k，即可实现为7个cpu共分配72k内存、从而提高内存利用率的目的。

Linux在为NUMA结构系统的cpu分配专用内存时，通过不断降低分配单位，确立一个合适的最终分配单位(unit per allocation-upa)，确保为每个NUMA节点分配的总的内存为最小单位整数倍的同时，有较高的内存利用率。

分配专用内存分配

对于多核系统，每个cpu的工作状态可能有所不同，进行的工作也有所差别，需要用cpu专用变量（per_cpu variables）保存一些特殊变量，以便与其它cpu进行区别。由于专用内存仅供一个cpu使用，大多数情况下，每个cpu访问其自身的专用变量时无需进行锁定。Linux提供了宏定义#define DEFINE_PER_CPU(type, name) DEFINE_PER_CPU_SECTION(type, name, “”)，其作用是定义cpu专用变量。每个cpu的专用变量都存储在自身的专用存储区内，该区域通过函数setup_per_cpu_areas()进行分配。对于单核系统或对称多核系统，该函数定义在文件git/mm/percpu.c文件中，对于NUMA系统，该函数定义在git/arch/yourprocessorname/mm/numa.c文件中（如git/arch/arm64/mm/numa.c）。

单核系统的函数定义为：

void __init setup_per_cpu_areas(void)
{
	const size_t unit_size = roundup_pow_of_two(
		max_t(size_t, PCPU_MIN_UNIT_SIZE, PERCPU_DYNAMIC_RESERVE));
	struct pcpu_alloc_info *ai;
	void *fc;

	ai = pcpu_alloc_alloc_info(1, 1);
	fc = memblock_alloc_from(unit_size, PAGE_SIZE, __pa(MAX_DMA_ADDRESS));
	if (!ai || !fc)
		panic("Failed to allocate memory for percpu areas.");
	kmemleak_free(fc);
	ai->dyn_size = unit_size;
	ai->unit_size = unit_size;
	ai->atom_size = unit_size;
	ai->alloc_size = unit_size;
	ai->groups[0].nr_units = 1;
	ai->groups[0].cpu_map[0] = 0;
	pcpu_setup_first_chunk(ai, fc);
	pcpu_free_alloc_info(ai);
}

对称多核系统的函数定义为：

void __init setup_per_cpu_areas(void)
{
	unsigned long delta;
	unsigned int cpu;
	int rc;
	rc = pcpu_embed_first_chunk(PERCPU_MODULE_RESERVE, PERCPU_DYNAMIC_RESERVE, 
PAGE_SIZE, NULL, pcpu_dfl_fc_alloc, pcpu_dfl_fc_free);
	if (rc < 0)
		panic("Failed to initialize percpu areas.");
	delta = (unsigned long)pcpu_base_addr - (unsigned long)__per_cpu_start;
	for_each_possible_cpu(cpu)
		__per_cpu_offset[cpu] = delta + pcpu_unit_offsets[cpu];
}

Arm64支持NUMA结构，函数的定义为：

void __init setup_per_cpu_areas(void)
{
	unsigned long delta;
	unsigned int cpu;
	int rc;

	rc = pcpu_embed_first_chunk(PERCPU_MODULE_RESERVE, PERCPU_DYNAMIC_RESERVE, 
						PAGE_SIZE, NULL, pcpu_dfl_fc_alloc, pcpu_dfl_fc_free);
	if (rc < 0)
		panic("Failed to initialize percpu areas.");
	delta = (unsigned long)pcpu_base_addr - (unsigned long)__per_cpu_start;
	for_each_possible_cpu(cpu)
		__per_cpu_offset[cpu] = delta + pcpu_unit_offsets[cpu];
}

为了便于内存管理，Linux为专用内存分配程序定义了结构体struct pcpu_alloc_info和struct pcpu_group_info，其中，结构体pcpu_alloc_info保存有各种内存大小、内存分组及内存分配单位等信息，定义为：

struct pcpu_alloc_info {
	size_t		static_size;
	size_t		reserved_size;
	size_t		dyn_size;
	size_t		unit_size;
	size_t		atom_size;
	size_t		alloc_size;
	size_t		__ai_size;
	int		nr_groups;
	struct pcpu_group_info	groups[];
}

unit_size为分配给单个cpu的内存字节数，alloc_size为实际分配的内存字节数。其余各个域的意义比较明显，这里不一一介绍。

结构体pcpu_group_info的定义为：

struct pcpu_group_info {
	int			nr_units;
	unsigned long	base_offset;	
	unsigned int		*cpu_map;
};

它保存cpu的分组情况，其中nr_units为节点实际分配的upa数目。Base_offset保存为节点分配的内存的偏移量，其中第一个节点的偏移量为0。cpu_map为一数组，记录当前节点的所有cpu逻辑编号。

了解专用内存的一些基本知识后，就比较容易理解专用内存的分配过程。对单核系统，函数setup_per_cpu_areas()首先通过调用函数pcpu_alloc_alloc_info(int nr_groups, int nr_units)为结构体pcpu_alloc_info和pcpu_group_info本身分配内存，其中nr_groups为NUMA节点数，nr_units为upa数目。很显然，单核系统的nr_groups和nr_inits均等于1。

在为两个结构体分配内存后，setup_per_cpu_areas()直接调用我们在前面遇到的memblock内存分配器分配足够的cpu专用内存。

通过对比对称多核系统和NUMA系统的setup_per_cpu_areas()函数可以发现，两者的函数定义基本相同，所不同的是传递给内存分配函数pcpu_embed_first_chunk()的最后三个参数有所不同。对称多核系统通过pcpu_embed_first_chunk(PERCPU_MODULE_RESERVE, PERCPU_DYNAMIC_RESERVE, PAGE_SIZE, NULL, pcpu_dfl_fc_alloc, pcpu_dfl_fc_free)方式调用，而NUMA系统通过pcpu_embed_first_chunk(PERCPU_MODULE_RESERVE, PERCPU_DYNAMIC_RESERVE, PAGE_SIZE, pcpu_cpu_distance, pcpu_fc_alloc, pcpu_fc_free)方式调用。pcpu_embed_first_chunk()函数的最后三个参数cpu_distance_fn、 alloc_fn、 free_fn为回调函数，分别用于确定各个cpu与不同的NUMA节点的距离、分配内存及释放内存。内存分配及释放函数为memblock内存分配和释放函数(memblock_alloc_try_nid及memblock_free)。由于对称多核系统所有cpu与内存结点的距离相同，即只有一个NUMA结点，因此，传递给pcpu_embed_first_chunk()函数的距离确定回调函数为空。

通过对pcpu_embed_first_chunk()分析可以发现，对于对称多核系统，pcpu_embed_first_chunk()的作用就是调用函数pcpu_build_alloc_info()确定每个CPU分配的单位个数（upa-units per allocation）、为结构体pcpu_alloc_info和pcpu_group_info分配内存（通过调用函数pcpu_alloc_alloc_info()）并进行初始化，最后通过调用回调函数alloc_fn()分配专用内存。在调用pcpu_alloc_alloc_info()时，nr_groups = 1，nr_units 为所需分配的单位数（upa的倍数）。

NUMA系统传递给pcpu_embed_first_chunk()的回调函数除去内存分配函数和内存释放函数外，还有一个函数cpu_distance_fn()，用以确定cpu与内存各个结点的距离。pcpu_build_alloc_info()利用该函数把cpu分成多个组（事实上是把访问同一个NUMA节点的cpu分在同一组）。

pcpu_build_alloc_info()首先确定upa的最大值，然后利用cpu_distance_fn()对CPU核芯进行分组，接着寻找一个最优的upa值，之后通过函数pcpu_alloc_alloc_info()为结构体pcpu_alloc_info和pcpu_group_info分配内存，最后初始化前述的两个结构体。

在函数pcpu_build_alloc_info()返回后，pcpu_embed_first_chunk()调用回调函数alloc_fn()，依次为各个组分配专用内存，分配的内存地址存放在数组areas中。如果内存分配回调函数运行失败，通过回调函数free_fn()，可以释放已分配的内存。

在分配了专用内存之后，如果硬件架构为为对称多核结构或NUMA结构， setup_per_cpu_areas()函数还要初始化数组__per_cpu_offset的内容，以确定各个cpu的per_cpu变量在专用内存中的位置。
